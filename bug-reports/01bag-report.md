# 🐞 Баг-репорт: кнопка «Выйти» не отображается в профиле

**Дата:** 2026-07-05  
**Автор:** Разработчик  
**Статус:** ✅ Исправлено  
**Приоритет:** Средний (мешает пользователю выйти из аккаунта)

---

## 1. Краткое описание

На экране **Профиль** (SCR-007) кнопка **«Выйти»** не отображалась для пользователя. Это делало невозможным выход из аккаунта и переход на экран авторизации.

---

## 2. Симптомы и шаги воспроизведения

**Симптом:**
- Пользователь переходит на вкладку «Профиль» (нижняя навигация).
- Интерфейс профиля отображается корректно: телефон, статус лояльности, контакты.
- Кнопка «Выйти» отсутствует .

**Шаги для воспроизведения:**
1. Войти в приложение (демо-код 1234).
2. Перейти на вкладку **«Профиль»**.
3. Прокрутить экран до конца.
4. Кнопки «Выйти» нет.

---

## 3. Ожидаемое поведение

В верхней части экрана **всегда** должна отображаться кнопка **«Выйти»**, которая при нажатии:
- очищает JWT-токен,
- возвращает пользователя на экран авторизации (SCR-001).

---

## 4. Фактическое поведение

- Кнопка **«Выйти»** не отображалась на экране профиля.
- Единственный способ выйти — переустановить приложение или очистить данные вручную.

---

## 5. Причина (Root Cause)

В исходной реализации `ProfileScreen` использовался компонент `PullToRefreshBox`, который оборачивал содержимое. При этом структура `Column` с `Spacer(Modifier.weight(1f))` была вложена внутрь него и не могла правильно занять всю высоту, из-за чего кнопка уходила за пределы видимой области.

Кроме того, была **логическая ошибка**: кнопка «Выйти» выводилась **только** при условии `client != null`. Если профиль не загружался или была ошибка сети, кнопка не показывалась (вместо неё отображался `ErrorState`). Но даже при успешной загрузке она могла быть скрыта из-за недостатка высоты.

---

## 6. Решение

### 6.1. Что было сделано

1. **Удалён `PullToRefreshBox`** — заменён на обычный `Column` с кнопкой обновления в `TopAppBar`.
2. **Явно указан `Modifier.fillMaxSize()`** для корневого контейнера, чтобы он занимал всю высоту экрана.
3. **Кнопка «Выйти» всегда внизу** — используется `Spacer(Modifier.weight(1f))`, чтобы прижать кнопку к нижнему краю.
4. **Дублирование выхода в TopAppBar** — добавлена иконка **Logout** (🚪) в правой части верхней панели, как альтернативный способ выхода.
5. **Исправлена логика видимости** — теперь кнопка отображается **всегда**, когда загружен профиль, и даже при ошибке доступен выход через иконку в TopAppBar.

### 6.2. Итоговые изменения в коде

**Было (проблемный фрагмент):**
```kotlin
Scaffold(...) { padding ->
    PullToRefreshBox(...) {
        Column(Modifier.padding(padding)) {
            // контент профиля
            Spacer(Modifier.weight(1f))
            ApexDestructiveButton(text = "Выйти", onClick = { ... })
        }
    }
}
