# 🐞 Баг-репорт: общий таймер отправки SMS для всех аккаунтов

**Дата:** 2026-07-05  
**Автор:** Разработчик  
**Статус:** ✅ Исправлено  
**Приоритет:** Высокий (критический для UX, нарушает логику работы с несколькими пользователями)

---

## 1. Краткое описание

Таймер повторной отправки SMS-кода (60 секунд) работает **глобально для всех аккаунтов**. Если пользователь запросил код, вышел и зашёл с другим номером телефона, то при попытке повторно запросить код система сообщает, что нужно подождать, хотя это другой аккаунт и другой номер.

---

## 2. Симптомы и шаги воспроизведения

**Симптомы:**
- После отправки SMS с одного номера таймер блокирует отправку для любого другого номера.
- При попытке получить код для другого аккаунта появляется сообщение: «Повторная отправка доступна через N секунд».
- Таймер не сбрасывается при выходе из аккаунта или переключении пользователя.

**Шаги для воспроизведения:**
1. Открыть приложение (экран авторизации).
2. Ввести номер +7 (999) 123-45-67, нажать «Получить SMS-код» → таймер запускается на 60 секунд.
3. Выйти из приложения или перезапустить его.
4. Ввести другой номер +7 (999) 987-65-43, нажать «Получить SMS-код».
5. Появляется сообщение: «Повторная отправка доступна через 45 секунд» (остаток времени от предыдущего запроса).

**Ожидаемое поведение:** таймер должен сбрасываться при смене номера телефона или при входе в другой аккаунт, так как отправка кода привязана к конкретному номеру.

---

## 3. Ожидаемое поведение

- Таймер повторной отправки должен быть **индивидуальным для каждого номера телефона**.
- При смене номера телефона на экране авторизации таймер должен сбрасываться.
- Каждый новый запрос кода для нового номера должен запускать собственный таймер.
- Если пользователь вышел из аккаунта, все данные сессии (включая таймер) должны очищаться.

---

## 4. Фактическое поведение

- Таймер хранится в единственном экземпляре `timerSeconds` в `RegistrationScreen`.
- Момент отправки (`smsSentAt`) хранится в `ApexRepository` как одно поле, общее для всех запросов.
- При смене номера таймер не сбрасывается, и система продолжает отсчитывать оставшееся время от предыдущего запроса.
- Даже после выхода из аккаунта (вызов `logout()`) таймер не очищается.

---

## 5. Причина (Root Cause)

**Логическая ошибка в архитектуре хранения состояния:**

В `ApexRepository` использовалась одна переменная `smsSentAt: Instant?`, которая хранила время последней отправки для **любого номера**:

```kotlin
// Было (проблемный код)
private var smsSentAt: Instant? = null

suspend fun sendSmsCode(phone: String) {
    // ...
    val lastSent = smsSentAt
    if (lastSent != null && Instant.now().isBefore(lastSent.plusSeconds(60))) {
        val retry = 60 - (Instant.now().epochSecond - lastSent.epochSecond)
        throw ApiException(ApiErrorCode.RATE_LIMITED, "Повторная отправка доступна через $retry секунд", retry.toInt())
    }
    // ...
    smsSentAt = Instant.now()
}
