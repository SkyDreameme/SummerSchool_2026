# Описание предметной области — «Картинг-центр „Апекс“»
## Клиентское мобильное приложение (текущая поставка)

---

## 1. Контекст бизнеса

**Картинг-центр «Апекс»** — уличный картинг-центр, работающий третий год. Клиенты катаются группами (заездами) по трассе; один заезд длится 15–20 минут, перед стартом проводится инструктаж.

**Ключевая проблема:** запись ведётся вручную через Telegram и маркерную доску. В выходные возникает хаос: двойные записи, клиенты приезжают к уехавшему заезду, маршалы и карта простаивают.

**Цель текущей поставки:** дать клиентам возможность самостоятельно видеть свободные слоты и бронировать заезды через мобильное приложение, сняв операционную нагрузку с владельца.

---

## 2. Границы системы

### В скоупе текущей поставки
- Клиентское мобильное приложение (роль «Клиент»).
- Контракт API между приложением и существующим бэкендом.

### Вне скоупа (уже реализовано в инфраструктуре)
- Бэкенд, хранение данных, формирование расписания.
- Интерфейс маршала.
- Административная панель владельца.
- Атомарная проверка свободных мест при бронировании — ответственность бэкенда.

Бэкенд рассматривается как **black-box источник истины**: приложение полагается на его ответы и корректно обрабатывает отказы (в т.ч. «мест нет»).

---

## 3. Роли

В рамках текущей поставки в приложении присутствует **только одна роль**:

| Роль | Описание |
|---|---|
| **Клиент** | Физическое лицо, регистрируется по номеру телефона, просматривает расписание, бронирует заезды, оплачивает депозит, получает уведомления, оценивает маршала. |

Маршал и Владелец/Администратор — роли существуют в системе, но их интерфейсы и сценарии **вне скоупа** текущей поставки.

---

## 4. Ключевые сущности

### 4.1. Клиент
Физическое лицо — пользователь приложения.

| Атрибут | Описание |
|---|---|
| `phone` | Номер телефона (единственный идентификатор, используется для SMS-входа). |
| `loyaltyStatus` | Текущий статус лояльности: `None` / `Bronze` / `Silver`. |
| `discountPercent` | Актуальная скидка (0% / 10% / 20%). |

### 4.2. Конфигурация трассы
Одна из двух доступных раскладок трассы:
- **Короткая** — для новичков, до 8 клиентов в заезде.
- **Длинная** — для опытных, до 14 клиентов.

Выбирается клиентом при бронировании. Валидация «опытности» клиента **не производится** — доверяем выбору.

### 4.3. Заезд (слот)
Единица расписания, сформированная бэкендом.

| Атрибут | Описание |
|---|---|
| `id` | Уникальный идентификатор. |
| `startsAt` | Дата и время старта. |
| `endsAt` | Расчётное время окончания (с учётом инструктажа). |
| `trackConfiguration` | `Short` / `Long`. |
| `marshalId` | Назначенный маршал-инструктор. |
| `totalSeats` | Максимальное количество клиентов (8 или 14). |
| `availableSeats` | Свободных мест на момент запроса. |
| `priceWithRental` | Цена с прокатной экипировкой. |
| `priceOwnGear` | Цена со своей экипировкой. |
| `status` | `Open` / `CancelledByCenter`. |
| `cancellationReason` | Заполняется, если `status = CancelledByCenter`. |

### 4.4. Бронь
Связь «клиент ↔ заезд». Создаётся приложением, хранится и контролируется бэкендом.

| Атрибут | Описание |
|---|---|
| `id` | Уникальный идентификатор. |
| `clientId` | Клиент. |
| `slotId` | Заезд. |
| `gearOption` | `Rental` / `Own`. |
| `finalPrice` | Итоговая цена с учётом скидки лояльности и выбора экипировки. |
| `depositLink` | Ссылка на перевод по номеру телефона (генерируется приложением/бэкендом). |
| `paymentStatus` | `Pending` / `Paid` / `Expired`. |
| `status` | См. раздел 5. |
| `rated` | Флаг: оставлена ли оценка маршалу. |

**Ограничение:** 1 клиент = 1 бронь в одном заезде. Групповые брони **не поддерживаются**.

### 4.5. Маршал
Инструктор, ведущий заезд. В приложении отображается только как атрибут слота (имя). Публичный рейтинг маршала **не выводится** — оценки используются только для внутренней аналитики владельца.

### 4.6. Оценка маршала

| Атрибут | Описание |
|---|---|
| `slotId` | Заезд, к которому относится. |
| `clientId` | Клиент, оценивающий. |
| `rating` | Целое число от 1 до 5 («звёздочки»). |

Текстовый комментарий **не предусмотрен**. Запрос на оценку инициируется push-уведомлением **через 1 час** после расчётного окончания заезда.

## Итоговое резюме

По результатам уточнений зафиксированы следующие ключевые решения, которые вошли в `domain-description.md`:

1. **Скоуп роли:** только клиент. Маршал и админ — вне текущей поставки.
2. **Авторизация:** единственная — по SMS на номер телефона.
3. **Бронирование:** 1 клиент = 1 бронь, без групповых заявок, без валидации опыта, без waivers.
4. **Оплата:** через ссылку на перевод по номеру телефона (без встроенного эквайринга).
5. **Отмены:** жёсткое правило «2 часа»; лист ожидания не делается.
6. **Оценки маршалам:** только 1–5 звёзд, без текста, без публичного рейтинга.
7. **Уведомления:** 3 типа push (24ч / 2ч до старта, 1ч после окончания). Без action-кнопок.
8. **Лояльность:** 3 статуса (нет / бронза 10% / серебро 20%), определяется бэкендом.
9. **История:** раздел «Мои заезды» обязателен.

### Правила переходов
- **Создана → Подтверждена**: после факта оплаты депозита (флаг выставляется бэкендом).
- **Подтверждена → Состоялась**: автоматически по факту прошедшего времени окончания заезда.
- **Подтверждена → Отменена клиентом**: только если до старта ≥ 2 часов. При меньшем таймфрейме API отклоняет запрос, кнопка «Отменить» в приложении становится неактивной.
- **Подтверждена → Отменена центром**: инициируется бэкендом (погода, решение владельца). Повторная запись на этот же слот запрещена.

## 6. Ключевые бизнес-правила

### 6.1. Бронирование
- Клиент выбирает слот из расписания на **7 дней** вперёд (по умолчанию); больший горизонт — через фильтр дат.
- Клиент выбирает конфигурацию трассы и вариант экипировки.
- Бэкенд отдаёт две цены (`priceWithRental`, `priceOwnGear`); приложение применяет скидку лояльности к выбранной.
- **1 человек = 1 бронь.** Групповые брони не поддерживаются.
- Гарантию отсутствия двойных броней обеспечивает бэкенд (атомарная проверка).

### 6.2. Оплата
- Онлайн-эквайринг **не подключается**.
- Оплата производится по **ссылке на перевод по номеру телефона**, которая генерируется в приложении после создания брони.
- Бронь считается подтверждённой после фиксации оплаты на стороне бэкенда.

### 6.3. Отмены
- Отмена клиентом за ≥ 2 часа до старта — бесплатно, место освобождается.
- Отмена клиентом за < 2 часа — API отклоняет операцию, в UI кнопка «Отменить» неактивна.
- Отмена центром — бронь получает статус «Отменена центром» с причиной; клиент получает push. Повторная запись на этот слот запрещена. Компенсация/возврат средств в рамках приложения **не реализуются** (решается вручную владельцем).

### 6.4. Лист ожидания
**Не реализуется.** Если мест нет — клиент видит соответствующий empty state.

### 6.5. Уведомления
- **Push за 24 часа** до старта — напоминание о заезде.
- **Push за 2 часа** до старта — финальное напоминание.
- **Push через 1 час** после окончания заезда — запрос оценки маршалу.
- Кнопки быстрого подтверждения («Я буду» / «Отменить») в напоминания **не добавляются**.
- Push об отмене центром — с указанием причины.

### 6.6. Программа лояльности
Статус и размер скидки определяются на стороне бэкенда и отдаются в профиле клиента:

| Статус | Скидка |
|---|---|
| Нет статуса | 0% |
| Бронза | 10% |
| Серебро | 20% |

Приложение отображает текущий статус и применяет скидку при расчёте `finalPrice`.

### 6.7. Авторизация
- Только по **номеру телефона через SMS-код**.
- Email, соцсети, пароли — не используются.

### 6.8. Возрастные и медицинские ограничения
**Не реализуются** в приложении: чекбоксы согласия, загрузка документов, цифровые waivers — вне скоупа.

### 6.9. История заездов
Раздел «Мои заезды» содержит:
- Активные брони (будущие заезды).
- Состоявшиеся заезды (история).
- Отменённые заезды (клиентом и центром).

## 7. Интеграции и внешние зависимости

| Интеграция | Назначение | Ответственная сторона |
|---|---|---|
| SMS-шлюз | Отправка кодов авторизации | Бэкенд |
| Push-уведомления (APNs/FCM) | Напоминания, запросы оценок, уведомления об отменах | Бэкенд (инициирует), приложение (получает) |
| Генератор ссылок на перевод | Формирование `depositLink` для оплаты | Бэкенд / приложение (уточняется в контракте API) |
| Бэкенд «Апекса» | Источник истины по слотам, броням, клиентам, лояльности | Black-box |
