3. Назначение и контекст
   3.1. Краткое описание
   Детальная карточка слота с итоговой ценой с учётом скидки лояльности. Предпоследний шаг перед бронированием.

3.2. Пользовательская цель
Клиент хочет увидеть итоговую цену с учётом своей скидки, чтобы понимать, сколько заплатит.

3.3. Бизнес-ценность
Прозрачность цены (00-foundations §2.5) → снижение отказа на шаге бронирования.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: тап по карточке слота на SCR-002

4. Навигация
   4.1. Входящая навигация
   | Источник | Триггер | Условие |
   | --- | --- | --- |
   | SCR-002 | Тап по карточке слота | availableSeats > 0 и status == Open |

4.2. Исходящая навигация

4.3. Системные жесты
Swipe-back: разрешён → возврат на SCR-002.

5. Данные
   5.1. Локальные модели
   | Сущность | Атрибуты | RW/RO | Источник |
   | --- | --- | --- | --- |
   | Slot | id, startsAt, endsAt, trackConfiguration, marshal, availableSeats, priceWithRental, priceOwnGear, status, cancellationReason | RO | GET /slots/{id} |
   | Client | loyaltyStatus, discountPercent | RO | GET /clients/me |
   | Computed: finalPrice | price × (1 − discountPercent/100) | Computed | Бэкенд (FR-7) |

5.2. API-эндпоинты
| Метод | Endpoint | Назначение | Обрабатываемые коды |
| --- | --- | --- | --- |
| GET | api/slots/api.yaml → /slots/{slotId} | Детали слота | 200, 404 |
| GET | api/profile/api.yaml → /clients/me | Профиль для скидки | 200, 401 |

5.3. Кэширование и актуальность
TTL кэша: не кэшируется.
Источник истины: бэкенд (NFR-1).

6. Структура экрана
   6.1. Блок: Заголовок слота
   Назначение: дата, время, конфигурация трассы
   Компоненты:
- `Date` + `Time` (startsAt)
- `TrackConfigLabel` (Short/Long) + иконка «ⓘ» → BS-004
- `MarshalName` (имя маршала, без рейтинга — FR-18)
  Данные: Slot.startsAt, Slot.trackConfiguration, Slot.marshal

6.2. Блок: Цены
Назначение: две базовые цены
Компоненты:
- `PriceRow` «С прокатной экипировкой — priceWithRental»
- `PriceRow` «Со своей экипировкой — priceOwnGear»
  Данные: Slot.priceWithRental, Slot.priceOwnGear

6.3. Блок: Итоговая цена и лояльность
Назначение: показать finalPrice с учётом скидки
Компоненты:
- `LoyaltyBadge` (None/Bronze/Silver)
- `FinalPrice` (крупно)
- `DiscountLine` «Скидка N% применена»
  Данные: Client.loyaltyStatus, Client.discountPercent, Computed finalPrice
  Видимость: всегда

6.4. CTA
| CTA | Тип | Активен при | Действие |
| --- | --- | --- | --- |
| «Забронировать» | primary | availableSeats > 0 и status == Open | push SCR-004 |
| «Забронировать» | disabled | availableSeats == 0 | Текст «Мест нет» |
| «Забронировать» | disabled | status == CancelledByCenter | Текст «Заезд отменён центром» (п. 25 из review) |

8. Логика и взаимодействия
   8.1. Инициализация
1. Запустить параллельно GET /slots/{id} и GET /clients/me.
2. Показать скелетон.
3. При успехе — отрисовать контент.
4. При ошибке — показать Error.

8.2. Happy path
1. Пользователь тапает по карточке слота.
2. Приложение загружает детали + профиль.
3. UI показывает итоговую цену с применённой скидкой.
4. Пользователь тапает «Забронировать» → SCR-004.

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |
| 1 | Скидка применяется только отображаемо; финальная цена — на бэкенде | FR-7, NFR-11 |
| 2 | Нет валидации «опытности» клиента | NFR-8 |
| 3 | Отображаем статус лояльности (расширение FR-7, п. 5 из review) | 00-foundations §2.5 |

11. Критерии приёмки (DoD)
- [ ] Итоговая цена показана с применённой скидкой.
- [ ] Статус лояльности отображается.
- [ ] CTA disabled при отсутствии мест или отмене центром.
- [ ] Переход на BS-004 работает.

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Слот отменён центром (status == CancelledByCenter) | CTA disabled, текст «Заезд отменён центром. Причина: {cancellationReason}» (п. 25 из review) |
    | 2 | Бэкенд вернул неизвестный статус лояльности (Gold) | Fallback на «Нет статуса», логирование (п. 26 из review) |
    | 3 | 404 при загрузке слота | Error state + CTA «К расписанию» |

12.1. Явные ограничения скоупа
- Нет валидации опыта (NFR-8).
- Нет визуальной карты (NFR-8).
- Нет публичного рейтинга маршала (FR-18).
