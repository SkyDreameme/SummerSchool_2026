3. Назначение и контекст
   3.1. Краткое описание
   Сценарий расширения диапазона дат расписания за пределы стандартных 7 дней.

3.2. Пользовательская цель
Как клиент, я хочу выбрать конкретную дату за пределами стандартных 7 дней, чтобы найти удобный слот.

3.3. Бизнес-ценность
Увеличение конверсии: клиент находит удобный слот даже в отдалённом будущем.

3.4. Контекст запуска
Точка входа: тап по «Расширить диапазон» в SCR-002
Сессия: авторизован
Сеть: обязательна

6. Граф переходов
   6.1. Happy path
1. Пользователь на SCR-002 тапает «Расширить диапазон».
2. Приложение → open BS-003.
3. Пользователь выбирает from и to в календаре.
4. Тапает «Применить».
5. Приложение → dismiss BS-003 + reload SCR-002 (GET /slots?from=...&to=...).
6. UI показывает слоты на новый диапазон.

6.2. Альтернативные пути
A1. На выбранную дату нет слотов
Условие входа: items.length == 0 после reload
Шаги:
1. Показать inline-предупреждение «На выбранную дату слотов нет» (п. 19 из review).
2. CTA «Сбросить фильтр» → возврат к 7 дням.
   Возврат в happy path: нет.

A2. Сеть упала при применении
Условие входа: ошибка сети на reload
Шаги:
1. Оставить предыдущий диапазон.
2. Показать toast «Не удалось применить фильтр» (п. 20 из review).
   Возврат в happy path: пользователь может повторить.

A3. Бэкенд не отдаёт признак наличия слотов на даты
Условие входа: BS-003 не может подсветить даты со слотами
Шаги:
1. Все даты кажутся доступными.
2. После применения — empty state в SCR-002 (п. 19 из review).
   Возврат в happy path: с шага 6 (A1).
