3. Назначение и контекст
   3.1. Краткое описание
   Сквозной сценарий отмены брони: детали брони → подтверждение → успех → возврат в список.

3.2. Пользовательская цель
Как клиент, я хочу отменить бронь не позднее чем за 2 часа до старта, чтобы освободить место.

3.3. Бизнес-ценность
Снижение нагрузки на администратора: клиент сам управляет бронью.

3.4. Контекст запуска
Точка входа: тап по CTA «Отменить» в SCR-006 (при canCancel == true)
Сессия: авторизован
Сеть: обязательна

5. Предусловия и постусловия
   5.1. Предусловия
- [ ] Клиент авторизован.
- [ ] Бронь в статусе Pending / Paid.
- [ ] now + 2h ≤ startsAt (canCancel == true от бэкенда).

5.2. Постусловия
- [ ] Бронь в статусе CancelledByClient.
- [ ] Клиент находится на SCR-005 (вкладка «Отменённые»).

5.3. Точки выхода
| Точка выхода | Условие | Финальное состояние |
| --- | --- | --- |
| Успех | 200 на POST /bookings/{id}/cancel | BS-005 → SCR-005 (вкладка «Отменённые») |
| Отмена пользователем | Тап «Оставить бронь» / swipe-down | Возврат к SCR-006 без изменений |
| Ошибка сети | Нет соединения | Inline-ошибка в BS-001 + CTA «Повторить» |
| Окно отмены закрылось | 422 CANCEL_WINDOW_CLOSED | Inline-ошибка «Время отмены истекло. Свяжитесь с администратором» |
| Бронь не найдена | 404 | Inline-ошибка «Бронь не найдена» |

6. Граф переходов
   6.1. Happy path
1. Пользователь на SCR-005 тапает по карточке активной брони.
2. Приложение → push SCR-006.
3. Пользователь видит CTA «Отменить» (canCancel == true).
4. Тапает «Отменить» → open BS-001.
5. Читает предупреждение о невозвратности.
6. Тапает «Отменить бронь» (деструктивное действие).
7. Приложение → POST /bookings/{id}/cancel → 200.
8. UI открывает BS-005 (success-шторка).
9. Пользователь тапает «К моим записям».
10. replace на SCR-005 (вкладка «Отменённые»).

6.2. Альтернативные пути
A1. Окно отмены закрылось пока пользователь думал
Условие входа: 422 CANCEL_WINDOW_CLOSED
Шаги:
1. Показать inline-ошибка в BS-001 «Время отмены истекло. Свяжитесь с администратором».
2. CTA «Закрыть» → dismiss BS-001, возврат к SCR-006.
   Возврат в happy path: нет, флоу завершён.

A2. Пользователь передумал
Условие входа: тап «Оставить бронь» / swipe-down
Шаги:
1. Закрыть BS-001 без запроса к бэкенду.
2. Возврат к SCR-006.
   Возврат в happy path: нет.

7. Используемые API
   | # | Метод | Endpoint | Назначение | Успешный код | Обрабатываемые ошибки |

   | 1 | GET | api/bookings/api.yaml → /bookings/{id} | Детали брони (canCancel) | 200 | 404 |

7.1. Последовательность вызовов
Вызов #1 — при открытии SCR-006.
Вызов #2 — после подтверждения в BS-001.

7.2. Идемпотентность и повторные попытки
- Вызов #2: идемпотентен (повторная отмена уже отменённой брони → 404 или 200). Повтор при сетевой ошибке — да.

9. Обработка ошибок
   9.1. Сетевые ошибки
   | Ситуация | Поведение |
   | --- | --- |
   | Нет соединения при старте флоу | Error state в SCR-006 |
   | Потеря связи на шаге отмены | Inline-ошибка в BS-001 + CTA «Повторить» |
   | Таймаут (> 10 с) | Error state |

9.2. Ошибки бэкенда
| Код | Смысл | Поведение |
| --- | --- | --- |
| 404 | Бронь не найдена | Inline-ошибка «Бронь не найдена» |
| 422 CANCEL_WINDOW_CLOSED | Окно отмены закрыто | Inline-ошибка «Время отмены истекло. Свяжитесь с администратором» |

9.3. Race conditions
| # | Сценарий | Обработка |
| --- | --- | --- |
| 1 | Окно отмены закрылось пока пользователь думал (п. 9 из review) | 422 CANCEL_WINDOW_CLOSED → inline-ошибка |
| 2 | Ровно за 2 часа до старта (п. 10 из review) | Условие now + 2h == startsAt выполнено включительно → кнопка активна |

10. Edge cases
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Окно отмены закрылось пока пользователь думал (п. 9 из review) | 422 → inline-ошибка «Время отмены истекло. Свяжитесь с администратором» |
    | 2 | Ровно за 2 часа до старта (п. 10 из review) | now + 2h == startsAt → условие выполнено включительно, кнопка активна |
    | 3 | canCancel == false (окно закрыто) | CTA «Отменить» disabled с тултипом «Отмена доступна не позднее чем за 2 часа до старта» (п. 3 из review §2) |
    | 4 | 404 (бронь удалена) | Inline-ошибка «Бронь не найдена» |
    | 5 | Сеть упала во время запроса | Inline-ошибка + CTA «Повторить» |
    | 6 | Двойной тап по «Отменить бронь» | CTA disabled во время запроса |
