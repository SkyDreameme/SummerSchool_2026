3. Назначение и контекст
   3.1. Краткое описание
   Сквозной сценарий оценки маршала: push через 1 час после заезда → модалка → оценка 1–5 звёзд → success-шторка.

3.2. Пользовательская цель
Как клиент, я хочу получить push с запросом оценки маршала через 1 час после окончания заезда, чтобы поделиться впечатлением.

3.3. Бизнес-ценность
Обратная связь для улучшения сервиса (внутренняя аналитика).

3.4. Контекст запуска
Точка входа: push-уведомление через 1 час после окончания заезда / тап по CTA «Оценить маршала» в SCR-006
Сессия: авторизован
Сеть: обязательна

5. Предусловия и постусловия
   5.1. Предусловия
- [ ] Бронь в статусе Completed.
- [ ] now ≥ endsAt (заезд завершился).
- [ ] rated == false (клиент ещё не оценил).
- [ ] Заезд не был отменён центром (status != CancelledByCenter).

5.2. Постусловия
- [ ] Оценка отправлена (1–5 звёзд).
- [ ] rated == true на бэкенде.
- [ ] Клиент увидел success-шторку.

5.3. Точки выхода
| Точка выхода | Условие | Финальное состояние |
| --- | --- | --- |
| Успех | 201 на POST /slots/{slotId}/ratings | BS-006 → SCR-005 / dismiss |
| Отмена пользователем | Swipe-down / тап «Закрыть» | Возврат к SCR-005 / dismiss |
| Ошибка сети | Нет соединения | Inline-ошибка в SCR-008 |
| Уже оценил | 409 ALREADY_RATED | Тост «Вы уже оценили этот заезд» + закрытие |

6. Граф переходов
   6.1. Happy path (через push)
1. Заезд завершился → бэкенд ждёт 1 час.
2. Бэкенд отправляет push-уведомление (информационное, без action-кнопок — FR-13).
3. Клиент тапает по push → open SCR-008 (модалка).
4. Приложение проверяет локально: rated == false.
5. Клиент выбирает 1–5 звёзд.
6. Тапает «Отправить оценку».
7. Приложение → POST /slots/{slotId}/ratings → 201.
8. UI открывает BS-006 (success-шторка).
9. Клиент тапает «Закрыть» → dismiss / pop на SCR-005.

6.2. Happy path (через SCR-006)
1. Клиент открывает SCR-006 (детали брони).
2. Бронь в статусе Completed, now ≥ endsAt, rated == false.
3. CTA «Оценить маршала» активна.
4. Тапает CTA → open SCR-008 (модалка).
5. Шаги 5–9 идентичны.

6.2. Альтернативные пути
A1. Клиент уже оценил (push-дубликат)
Условие входа: rated == true (локально или 409 от бэкенда)
Шаги:
1. Показать тост «Вы уже оценили этот заезд».
2. Закрыть модалку.
   Возврат в happy path: нет.

A2. Заезд отменён центром
Условие входа: status == CancelledByCenter
Шаги:
1. Push не приходит (п. 12 из review).
2. Если открыли из SCR-006 — CTA «Оценить маршала» скрыта.
   Возврат в happy path: нет.

A3. Заезд ещё не завершился
Условие входа: now < endsAt
Шаги:
1. CTA «Оценить маршала» в SCR-006 скрыта (п. 13 из review).
2. Push не приходит.
   Возврат в happy path: нет.
