3. Назначение и контекст
   3.1. Краткое описание
   Сквозной сценарий бронирования: выбор слота → детали → выбор экипировки → создание брони → оплата депозита во внешнем банке → возврат в приложение.

3.2. Пользовательская цель
Как клиент, я хочу забронировать место и оплатить депозит, чтобы гарантировать себе участие в заезде.

3.3. Бизнес-ценность
Конверсия в бронь → доход центра (BR-1).

3.4. Контекст запуска
Точка входа: тап по карточке слота на SCR-002
Сессия: авторизован
Сеть: обязательна

5. Предусловия и постусловия
   5.1. Предусловия
- [ ] Клиент авторизован.
- [ ] На целевом слоте availableSeats > 0 и status == Open.
- [ ] У клиента нет активной брони на этот слот (FR-5).

5.2. Постусловия
- [ ] Бронь создана в статусе Pending.
- [ ] depositLink сгенерирован бэкендом.
- [ ] Клиент перешёл во внешний банк для оплаты.
- [ ] После оплаты — статус Paid (обновляется бэкендом).

5.3. Точки выхода
| Точка выхода | Условие | Финальное состояние |
| --- | --- | --- |
| Успех | 201 на POST /bookings | BS-002 → External |
| Отмена пользователем | Swipe-back на SCR-003 / SCR-004 | Возврат без создания брони |
| Ошибка сети | Нет соединения | Inline-ошибка + CTA «Повторить» |
| Мест нет (race) | 409 NO_SEATS | Inline-ошибка + CTA «Выбрать другое время» → SCR-002 |
| Бронь уже существует | 409 BOOKING_ALREADY_EXISTS | Inline-ошибка + CTA «К моим записям» → SCR-005 |
| Слот отменён центром | 410 SLOT_CANCELLED | Inline-ошибка с причиной + CTA «Выбрать другое время» |

6. Граф переходов
   6.1. Happy path
1. Пользователь на SCR-002 тапает по карточке слота.
2. Приложение → push SCR-003.
3. Пользователь видит итоговую цену со скидкой → тапает «Забронировать».
4. Приложение → push SCR-004.
5. Пользователь выбирает экипировку (Rental / Own) → тапает «Забронировать место».
6. Приложение → POST /bookings → 201 + Booking (с depositLink).
7. UI открывает BS-002 (success-шторка).
8. Пользователь тапает «Оплатить депозит» → open URL (External).
9. Клиент оплачивает во внешнем банке.
10. Возврат в приложение → pull-to-refresh в SCR-005 → статус Paid.

6.2. Альтернативные пути
A1. Race condition: мест нет во время создания
Условие входа: 409 NO_SEATS
Шаги:
1. Показать inline-ошибка «Мест на этот заезд больше нет».
2. CTA «Выбрать другое время» → pop до SCR-002.
   Возврат в happy path: нет, флоу завершён.

A2. Бронь уже существует
Условие входа: 409 BOOKING_ALREADY_EXISTS
Шаги:
1. Показать inline-ошибка «У вас уже есть бронь на этот заезд».
2. CTA «К моим записям» → replace на SCR-005 (вкладка «Активные»).
   Возврат в happy path: нет.

A3. Слот отменён центром во время создания
Условие входа: 410 SLOT_CANCELLED
Шаги:
1. Показать inline-ошибка «Заезд был отменён центром. Причина: {cancellationReason}».
2. CTA «Выбрать другое время» → pop до SCR-002.
   Возврат в happy path: нет.

A4. Клиент оплатил, но бэкенд не зафиксировал
Условие входа: возврат из External, статус всё ещё Pending
Шаги:
1. В BS-002 / SCR-006 показана инструкция «Если оплата прошла, но статус не изменился — свяжитесь с администратором».
2. Pull-to-refresh в SCR-005.
   Возврат в happy path: при следующем pull-to-refresh / push-инвалидации.

7. Используемые API
   | # | Метод | Endpoint | Назначение | Успешный код | Обрабатываемые ошибки |
   | --- | --- | --- | --- | --- | --- |
   | 1 | GET | api/slots/api.yaml → /slots/{slotId} | Детали слота | 200 | 404 |
   | 2 | GET | api/profile/api.yaml → /clients/me | Профиль для скидки | 200 | 401 |
   | 3 | POST | api/bookings/api.yaml → /bookings | Создание брони | 201 | 400, 409, 410 |

7.1. Последовательность вызовов
Вызовы #1 и #2 — параллельно на SCR-003.
Вызов #3 — на SCR-004, после тапа по CTA.

7.2. Идемпотентность и повторные попытки
- Вызов #3: не идемпотентен (создаёт бронь). Повтор при сетевой ошибке — да, но с проверкой 409 BOOKING_ALREADY_EXISTS (если бронь уже создана на предыдущей попытке).

9. Обработка ошибок
   9.1. Сетевые ошибки
   | Ситуация | Поведение |
   | --- | --- |
   | Нет соединения при старте флоу | Error state в SCR-003 |
   | Потеря связи на шаге создания брони | Inline-ошибка + CTA «Повторить» |
   | Таймаут (> 10 с) | Error state |

9.2. Ошибки бэкенда
| Код | Смысл | Поведение |
| --- | --- | --- |
| 400 | Некорректный запрос | Inline-ошибка с текстом из message |
| 409 NO_SEATS | Мест нет (race) | Inline-ошибка + CTA «Выбрать другое время» |
| 409 BOOKING_ALREADY_EXISTS | Бронь уже есть | Inline-ошибка + CTA «К моим записям» |
| 410 SLOT_CANCELLED | Слот отменён центром | Inline-ошибка с причиной + CTA «Выбрать другое время» |

    | 9 | Двойная оплата (п. 31 из review) | Out of Scope (задача бэкенда) |
    | 10 | Изменение конфигурации после создания брони (п. 22 из review) | Out of Scope (нужна новая бронь) |

10.1. Явные ограничения скоупа
- Нет онлайн-эквайринга (NFR-5).
- Нет групповых броней (NFR-6).
- Изменить конфигурацию после создания брони нельзя — нужна новая бронь (п. 22 из review).
- Частичная / двойная оплата — вне скоупа приложения (п. 30–31 из review).
