3. Назначение и контекст
   3.1. Краткое описание
   Фоновый сценарий: бэкенд отменяет заезд (например, из-за погоды) → клиент получает push → открывает детали брони → видит причину отмены.

3.2. Пользовательская цель
Как клиент, я хочу узнать причину отмены заезда центром, чтобы понимать, почему заезд не состоялся.

3.3. Бизнес-ценность
Прозрачность → снижение обращений в поддержку.

3.4. Контекст запуска
Точка входа: push-уведомление от бэкенда
Сессия: авторизован
Сеть: обязательна (для загрузки деталей)

5. Предусловия и постусловия
   5.1. Предусловия
- [ ] Бронь в статусе Paid.
- [ ] Бэкенд принял решение об отмене (например, погода).

5.2. Постусловия
- [ ] Бронь в статусе CancelledByCenter.
- [ ] cancellationReason заполнен.
- [ ] Клиент увидел причину отмены.

5.3. Точки выхода
| Точка выхода | Условие | Финальное состояние |
| --- | --- | --- |
| Успех | Клиент открыл SCR-006 | SCR-006 с отображением cancellationReason |
| Клиент проигнорировал push | — | Бронь в SCR-005 (вкладка «Отменённые») |
| Ошибка сети | Нет соединения | Error state в SCR-006 |

6. Граф переходов
   6.1. Happy path
1. Бэкенд отменяет заезд → статус CancelledByCenter + cancellationReason.
2. Бэкенд отправляет push-уведомление (информационное, без action-кнопок — FR-13).
3. Клиент тапает по push → open SCR-006 (с payload.bookingId).
4. Приложение → GET /bookings/{id} → 200 + BookingDetails (status = CancelledByCenter, cancellationReason).
5. UI показывает статус + причину отмены.
6. Клиент читает причину.

6.2. Альтернативные пути
A1. Клиент проигнорировал push
Условие входа: клиент не тапнул по push
Шаги:
1. При следующем открытии SCR-005 — бронь во вкладке «Отменённые».
2. Тап по карточке → SCR-006 с cancellationReason.
   Возврат в happy path: с шага 5.

A2. Push не пришёл (сбой APNs / FCM)
Условие входа: push не доставлен
Шаги:
1. Клиент видит бронь в SCR-005 (вкладка «Отменённые») при следующем pull-to-refresh.
2. Тап по карточке → SCR-006.
   Возврат в happy path: с шага 5.

10. Edge cases
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Push-дубликат (несколько push об одной отмене) | Каждый push открывает SCR-006 (одинаковое содержимое) |
    | 2 | Push не пришёл (сбой APNs / FCM) | Бронь видна в SCR-005 при pull-to-refresh |
    | 3 | cancellationReason пустой (баг бэкенда) | Fallback на «Причина не указана» + логирование |
    | 4 | Клиент открыл SCR-006 до получения push | Бронь всё ещё в Paid, отмена произойдёт позже |

10.1. Явные ограничения скоупа
- Нет авто-возвратов (NFR-9).
- Push — только информационный, без action-кнопок (FR-13).
