## Приложение A. Подсказки по заполнению

1. **Не оставляйте плейсхолдеры `<…>`** в финальной версии — замените их на конкретные значения.
2. **Traceability обязательна** — каждый FR/UC/US должен быть сопоставлен с блоком или CTA.
3. **Edge cases из `design-review.md`** — проверьте раздел 3 и перенесите релевантные сценарии в §12.
4. **Tone of Voice** — все пользовательские тексты проходят проверку по `00-foundations §5`.
5. **API-контракт** — все эндпоинты должны существовать в `api/<домен>/api.yaml`. Если эндпоинта нет — сначала обновить контракт.
6. **Согласование с BS** — если экран открывает шторку, для шторки должен быть отдельный документ `BS-XXX.md` по этому же шаблону.
