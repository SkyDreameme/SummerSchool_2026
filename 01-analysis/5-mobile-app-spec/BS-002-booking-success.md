3. Назначение и контекст
   3.1. Краткое описание
   Success-шторка после создания брони. Показывает depositLink и инструкцию по оплате.

3.2. Пользовательская цель
Клиент хочет получить ссылку на оплату депозита и инструкцию.

3.3. Бизнес-ценность
Прозрачный процесс оплаты → снижение неоплаченных броней.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: успешный ответ 201 на POST /bookings

6. Структура экрана
   6.1. Блок: Иконка успеха
   Назначение: визуальное подтверждение
   Компоненты: `SuccessIcon` (галочка)

6.2. Блок: Заголовок
Назначение: «Бронь создана»
Компоненты: `Title`

6.3. Блок: Сводка
Назначение: дата, время, конфигурация, экипировка, итоговая цена
Компоненты: `Summary`

6.4. Блок: Оплата депозита
Назначение: depositLink + инструкция
Компоненты:
- `DepositAmount`
- CTA «Оплатить депозит» → External
- CTA «Скопировать ссылку» (п. 9 из review)
- `Instruction` «Переведите сумму по ссылке, чтобы подтвердить бронь. Бронь автоматически перейдёт в статус Paid после поступления оплаты.»
- `Fallback` «Если оплата прошла, но статус не изменился — свяжитесь с администратором.» (п. 2 из review §3.1)

6.5. CTA

8. Логика и взаимодействия
   8.1. Инициализация
1. Получить Booking из SCR-004.
2. Отрисовать контент.

8.2. Happy path
1. Бронь создана → открывается BS-002.
2. Пользователь тапает «Оплатить депозит» → внешний банк.
3. После оплаты — возврат в приложение.
4. Бэкенд фиксирует оплату → статус Paid.
5. Пользователь видит Paid в SCR-005 (pull-to-refresh / push-инвалидация).

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |

| 2 | Нет онлайн-эквайринга | NFR-5 |
| 3 | Оплата фиксируется бэкендом | UC-4 |

- [ ] depositLink отображается.

- [ ] Инструкция показана.

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Клиент оплатил, но бэкенд не зафиксировал | Инструкция «свяжитесь с администратором» (п. 2 из review §3.1) |
    | 2 | Клиент потерял связь во внешнем банке | По возвращении — pull-to-refresh в SCR-005 (п. 3 из review §3.1) |
    | 3 | Частичная оплата | Бронь остаётся в Pending (п. 30 из review) |
    | 4 | Двойная оплата | Вне скоупа приложения (п. 31 из review) |

12.1. Явные ограничения скоупа
- Нет онлайн-эквайринга (NFR-5).
- Нет обработки частичной/двойной оплаты (п. 30-31 из review).
