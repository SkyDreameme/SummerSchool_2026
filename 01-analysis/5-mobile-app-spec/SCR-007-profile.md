3. Назначение и контекст
   3.1. Краткое описание
   Экран профиля: номер телефона, статус лояльности, размер скидки, контакты поддержки, выход.

3.2. Пользовательская цель
Клиент хочет видеть свой статус лояльности и размер скидки.

3.3. Бизнес-ценность
Прозрачность программы лояльности → мотивация к повторным визитам.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: тап по вкладке «Профиль»

6. Структура экрана
   6.1. Блок: Телефон
   Назначение: показать номер
   Компоненты: `PhoneLabel`
   Данные: Client.phone

6.2. Блок: Лояльность
Назначение: статус и скидка
Компоненты:
- `LoyaltyBadge` (None/Bronze/Silver)
- `DiscountLabel` «Скидка N%»
- (Опционально) `NextLevelHint` — текст об условиях перехода (п. 4 из review §1.1 — вне скоупа, т.к. нет источника правды)
  Данные: Client.loyaltyStatus, Client.discountPercent

6.3. Блок: Контакты
Назначение: связь с поддержкой (п. 28 из review)
Компоненты: `TextLink` «Связаться с поддержкой» → Telegram / tel:
Данные: захардкожены (или из remote config)

6.4. CTA
| CTA | Тип | Активен при | Действие |
| --- | --- | --- | --- |
| «Выйти» | destructive | всегда | Очистить JWT → SCR-001 |

   | Error | Сеть / 401 | «Нет соединения» / редирект на SCR-001 |
   | Success | 200 | Контент |

8. Логика и взаимодействия
   8.1. Инициализация
1. Запустить GET /clients/me.
2. Показать скелетон.
3. При успехе — отрисовать контент.
4. При 401 — редирект на SCR-001.

8.2. Happy path
1. Пользователь открывает вкладку «Профиль».
2. Приложение загружает профиль.
3. UI показывает телефон, статус, скидку.

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |
| 1 | Логика начисления статусов — на бэкенде | NFR-11 |
| 2 | Неизвестный статус лояльности — fallback «Нет статуса» | П. 26 из review |

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Неизвестный статус лояльности (Gold) | Fallback «Нет статуса» + логирование (п. 26 из review) |
    | 2 | 401 при загрузке | Редирект на SCR-001 |

12.1. Явные ограничения скоупа
- Смена номера телефона — вне скоупа (п. 18 из review).
- Удаление аккаунта — вне скоупа (п. 18 из review).
- Условия перехода на следующий уровень — вне скоупа (п. 4 из review §1.1, нет источника правды).
- Адрес, часы работы, правила трассы — вне скоупа (п. 29 из review).
