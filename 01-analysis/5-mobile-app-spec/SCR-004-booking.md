3. Назначение и контекст
   3.1. Краткое описание
   Экран подтверждения выбора экипировки и создания брони. После успеха — переход к BS-002 с depositLink.

3.2. Пользовательская цель
Клиент хочет выбрать экипировку и создать бронь, чтобы гарантировать себе участие.

3.3. Бизнес-ценность
Конверсия в бронь → доход центра.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: тап «Забронировать» на SCR-003

4. Навигация
   4.1. Входящая навигация
   | Источник | Триггер | Условие |
   | --- | --- | --- |
   | SCR-003 | CTA «Забронировать» | availableSeats > 0 и status == Open |

4.2. Исходящая навигация
| Действие | Целевой экран | Тип перехода |
| --- | --- | --- |
| Успех (201) | BS-002 | bottom sheet |
| BS-002 → «Оплатить депозит» | External (банк / СБП) | open URL |
| BS-002 → «К моим записям» | SCR-005 (вкладка «Активные») | replace на TabBar |
| Ошибка «мест нет» / «слот отменён» | SCR-002 | pop до корня + toast |
| Swipe-back | SCR-003 | pop |

4.3. Системные жесты
Swipe-back: разрешён → возврат на SCR-003 (бронь ещё не создана).

5. Данные
   5.1. Локальные модели
   | Сущность | Атрибуты | RW/RO | Источник |
   | --- | --- | --- | --- |
   | Slot | id, startsAt, trackConfiguration, priceWithRental, priceOwnGear | RO | Из SCR-003 |
   | BookingDraft | slotId, gearOption | RW | Локально |
   | Booking | id, status, finalPrice, depositLink | RO | POST /bookings (после успеха) |

5.2. API-эндпоинты
| Метод | Endpoint | Назначение | Обрабатываемые коды |
| --- | --- | --- | --- |
| POST | api/bookings/api.yaml → /bookings | Создание брони | 201, 400, 409 (NO_SEATS, BOOKING_ALREADY_EXISTS), 410 (SLOT_CANCELLED) |

5.3. Кэширование и актуальность
Источник истины: бэкенд (NFR-1).

6. Структура экрана
   6.1. Блок: Сводка брони
   Назначение: дата, время, конфигурация
   Компоненты: `Date`, `Time`, `TrackConfigLabel`
   Данные: Slot

6.2. Блок: Селектор экипировки
Назначение: выбор Rental / Own
Компоненты: `SegmentedControl` (Rental / Own)
Данные: BookingDraft.gearOption

6.3. Блок: Итоговая цена
Назначение: показать finalPrice с учётом скидки
Компоненты: `FinalPrice`, `DiscountLine`
Данные: Computed (на основе gearOption + Client.discountPercent)

6.4. CTA
| CTA | Тип | Активен при | Действие |
| --- | --- | --- | --- |
| «Забронировать место» | primary | всегда (доступность мест проверена на SCR-003) | POST /bookings |

7. Состояния экрана
   | Состояние | Триггер | Визуализация |
   | --- | --- | --- |
   | Loading (отправка) | Тап по CTA | Inline-спиннер на кнопке, CTA disabled |
   | Error (inline) | 409 NO_SEATS / BOOKING_ALREADY_EXISTS / 410 | Inline-ошибка + CTA «Выбрать другое время» |
   | Success | 201 | Открытие BS-002 |

7.1. Текст состояний
| Состояние | Заголовок | Подсказка |
| --- | --- | --- |
| 409 NO_SEATS | «Мест на этот заезд больше нет» | «Выберите другое время» |
| 409 BOOKING_ALREADY_EXISTS | «У вас уже есть бронь на этот заезд» | «Перейти к моим записям» |
| 410 SLOT_CANCELLED | «Заезд был отменён центром» | «{cancellationReason}. Выберите другое время» |
| 400 | «Не удалось создать бронь» | Текст из message |
| Сеть | «Нет соединения» | «Проверьте интернет и повторите» |

8. Логика и взаимодействия
   8.1. Инициализация
1. Получить Slot из SCR-003.
2. Инициализировать gearOption = Rental (по умолчанию).
3. Отрисовать контент.

8.2. Happy path
1. Пользователь выбирает экипировку.
2. Тапает «Забронировать место».
3. Приложение → POST /bookings → 201.
4. UI открывает BS-002 с depositLink.

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |
| 1 | 1 клиент = 1 бронь на слот | FR-5 |
| 2 | Атомарная проверка мест — на бэкенде | NFR-2 |
| 3 | Скидка лояльности применяется бэкендом | FR-7, NFR-11 |
| 4 | depositLink генерируется бэкендом | FR-8 |

10. Требования (traceability)
    10.1. Functional Requirements
    | ID | Формулировка | Как покрыто |
    | --- | --- | --- |
    | FR-3 | Выбор конфигурации | Блок сводки |
    | FR-4 | Выбор экипировки | Селектор |
    | FR-5 | 1 клиент = 1 бронь | Обработка 409 BOOKING_ALREADY_EXISTS |
    | FR-7 | Скидка лояльности | Блок итоговой цены |
    | FR-8 | depositLink | Передаётся в BS-002 |

10.2. Non-Functional Requirements
| ID | Формулировка | Как обеспечено |
| --- | --- | --- |
| NFR-1 | Бэкенд — источник истины | Все проверки на бэкенде |
| NFR-2 | Атомарная проверка мест | На бэкенде |
| NFR-5 | Нет онлайн-эквайринга | Переход по external URL |
| NFR-6 | Нет групповых броней | 1 бронь = 1 клиент |

11. Критерии приёмки (DoD)
- [ ] Happy path: создание брони → BS-002.
- [ ] Обработка 409 NO_SEATS / BOOKING_ALREADY_EXISTS.
- [ ] Обработка 410 SLOT_CANCELLED.
- [ ] Селектор экипировки меняет итоговую цену.

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Race condition: мест нет во время создания (409 NO_SEATS) | Inline-ошибка + CTA «Выбрать другое время» → SCR-002 (п. 7 из review) |
    | 2 | Бронь уже существует (409 BOOKING_ALREADY_EXISTS) | Inline-ошибка + CTA «К моим записям» → SCR-005 (п. 8 из review) |
    | 3 | Слот отменён центром во время создания (410 SLOT_CANCELLED) | Inline-ошибка с причиной + CTA «Выбрать другое время» (п. 6 из review) |
    | 4 | Двойной тап по CTA | CTA disabled во время запроса (идемпотентность) |
    | 5 | Сеть упала во время запроса | Inline-ошибка + CTA «Повторить» |

12.1. Явные ограничения скоупа
- Нет онлайн-эквайринга (NFR-5).
- Нет групповых броней (NFR-6).
- Изменить конфигурацию после создания брони нельзя — нужна новая бронь (п. 22 из review).
