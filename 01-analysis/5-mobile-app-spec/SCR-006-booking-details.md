3. Назначение и контекст
   3.1. Краткое описание
   Полная информация о брони: статус, детали заезда, возможность отмены и оценки маршала.

3.2. Пользовательская цель
Клиент хочет увидеть детали заезда и при необходимости отменить его или узнать причину отмены центром.

3.3. Бизнес-ценность
Снижение нагрузки на администратора: клиент сам видит статус и причину отмены.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: тап по карточке брони на SCR-005 / push-уведомление об отмене центром

6. Структура экрана
   6.1. Блок: Детали заезда
   Назначение: дата, время, конфигурация, экипировка, цена
   Компоненты: `Date`, `Time`, `TrackConfigLabel`, `GearLabel`, `FinalPrice`
   Данные: BookingDetails.slot, BookingDetails.gearOption, BookingDetails.finalPrice

6.2. Блок: Статус брони
Назначение: текущий статус + причина отмены (если есть)
Компоненты:
- `StatusBadge` (Pending / Paid / Expired / Completed / CancelledByClient / CancelledByCenter)
- `CancellationReason` (только для CancelledByCenter и CancelledByClient — если есть)
  Данные: BookingDetails.status, BookingDetails.cancellationReason

6.3. Блок: Оплата депозита (только для Pending)
Назначение: показать depositLink
Компоненты: `DepositLinkBlock`, CTA «Оплатить депозит» → External
Данные: BookingDetails.depositLink
Видимость: status == Pending

6.4. CTA
| CTA | Тип | Активен при | Действие |
| --- | --- | --- | --- |
| «Отменить» | destructive | canCancel == true (от бэкенда) | open BS-001 |
| «Оценить маршала» | secondary | status == Completed и rated == false и now ≥ endsAt | open SCR-008 (п. 13 из review) |
| «Оплатить депозит» | primary | status == Pending | open External |

8. Логика и взаимодействия
   8.1. Инициализация
1. Запустить GET /bookings/{id}.
2. Показать скелетон.
3. При успехе — отрисовать контент.
4. При ошибке — показать Error.

8.2. Happy path
1. Пользователь открывает детали брони.
2. Приложение загружает BookingDetails.
3. UI показывает статус, детали, CTA.

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |
| 1 | canCancel вычисляется бэкендом (now + 2h ≤ startsAt) | FR-10, NFR-1 |
| 2 | Все 6 статусов отображаются | FR-9, п. 2 из review (добавлен CancelledByClient) |
| 3 | CTA «Оценить маршала» активна только если rated == false и now ≥ endsAt | П. 13 из review |
| 4 | Если оплата прошла, но статус Pending — инструкция связаться с администратором | П. 2 из review (§3.1) |

11. Критерии приёмки (DoD)
- [ ] Все 6 статусов отображаются.
- [ ] CTA «Отменить» активна только при canCancel == true.
- [ ] CTA «Оценить маршала» учитывает rated и endsAt.
- [ ] cancellationReason показывается для CancelledByCenter.

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Статус CancelledByClient | Отображается, CTA «Отменить» скрыта (п. 2 из review) |
    | 2 | canCancel == false (окно закрыто) | CTA «Отменить» disabled с тултипом «Отмена доступна не позднее чем за 2 часа до старта» (п. 3 из review) |
    | 3 | Заезд отменён центром | CTA «Оценить маршала» скрыта (п. 12 из review) |
    | 4 | rated == true | CTA «Оценить маршала» скрыта (п. 11 из review) |
    | 5 | now < endsAt | CTA «Оценить маршала» скрыта (п. 13 из review) |
    | 6 | Pending долго не оплачивается | Инструкция «Если оплата прошла, но статус не изменился — свяжитесь с администратором» (п. 2 из review §3.1) |
    | 7 | Неизвестный статус | Fallback на «Отменённые» + логирование (п. 27 из review) |

12.1. Явные ограничения скоупа
- Нет авто-возвратов (NFR-9).
- Нет смены конфигурации после создания брони (п. 22 из review).
