3. Назначение и контекст
   3.1. Краткое описание
   Главный экран приложения — список слотов на 7 дней вперёд с возможностью расширения горизонта. Точка входа в сценарий бронирования.

3.2. Пользовательская цель
Клиент хочет просмотреть доступные слоты и выбрать удобное время заезда.

3.3. Бизнес-ценность
Основной конверсионный экран: от него стартует UC-3 (создание брони).

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна
Предшествующее действие: успешная авторизация / переключение на вкладку «Расписание»

4. Навигация
   4.1. Входящая навигация
   | Источник | Триггер | Условие |

4.3. Системные жесты
Swipe-back: неприменимо (корневой таб).
Swipe по дням: горизонтальный свайп переключает даты (7 дней по умолчанию).

5. Данные
   5.1. Локальные модели
   | Сущность | Атрибуты | RW/RO | Источник |
   | --- | --- | --- | --- |
   | Slot | id, startsAt, endsAt, trackConfiguration, availableSeats, priceWithRental, priceOwnGear, status | RO | GET /slots |
   | DateRange | from, to | RW | Локально (по умолчанию today…today+7) |

5.2. API-эндпоинты
| Метод | Endpoint | Назначение | Обрабатываемые коды |
| --- | --- | --- | --- |
| GET | api/slots/api.yaml → /slots | Список слотов с фильтром from/to | 200, 400 |

5.3. Кэширование и актуальность
TTL кэша: не кэшируется.
Стратегия: pull-to-refresh.
Источник истины: бэкенд (NFR-1).

6. Структура экрана
   6.1. Блок: Переключатель дат
   Назначение: горизонтальный список дат (7 дней по умолчанию)
   Компоненты:
- `DateChip` — дата + день недели, активная дата подсвечена
- `ButtonSecondary` «Расширить диапазон» → BS-003
  Данные: DateRange
  Видимость: всегда

6.2. Блок: Список слотов
Назначение: карточки слотов
Компоненты:
- `SlotCard` — время, конфигурация, две цены, индикатор мест
- При `availableSeats <= 0` — карточка с лейблом «Мест нет», некликабельна (FR-11)
- При `status == CancelledByCenter` — карточка с лейблом «Отменён центром» (п. 25 из review)
  Данные: Slot[]
  Видимость: всегда

6.3. CTA
| CTA | Тип | Активен при | Действие |
| --- | --- | --- | --- |
| Тап по карточке слота | — | availableSeats > 0 и status == Open | push SCR-003 |
| «Расширить диапазон» | secondary | всегда | open BS-003 |

7. Состояния экрана
   | Состояние | Триггер | Визуализация |
   | --- | --- | --- |
   | Loading (первичная) | Первый вход | Скелетон по форме карточек |
   | Loading (обновление) | Pull-to-refresh | Inline-спиннер |
   | Error | Сеть / 5xx / таймаут 10 с | Иллюстрация + «Повторить» |
   | Empty (нет слотов на диапазон) | items.length == 0 | «Пока нет доступных заездов» + «Загляните позже» |
   | Empty (на выбранную дату нет слотов после BS-003) | П. 19 из review | Inline-предупреждение «На выбранную дату слотов нет» |

7.1. Текст состояний
| Состояние | Заголовок | Подсказка |
| --- | --- | --- |
| Empty (нет слотов) | «Пока нет доступных заездов» | «Загляните позже — расписание обновляется» |
| Error | «Не удалось загрузить расписание» | «Повторите через минуту» |

8. Логика и взаимодействия
   8.1. Инициализация
1. Проверить авторизацию → при отсутствии: редирект на SCR-001.
2. Запустить GET /slots?from=today&to=today+7.
3. Показать скелетон.
4. При успехе — отрисовать карточки.
5. При ошибке — показать Error.

8.2. Happy path
1. Пользователь открывает вкладку «Расписание».
2. Приложение загружает слоты на 7 дней.
3. Пользователь свайпает/тапает по дате → список обновляется.
4. Пользователь тапает по карточке слота → SCR-003.

8.3. Бизнес-правила
| # | Правило | Обоснование |
| --- | --- | --- |
| 1 | Диапазон по умолчанию — 7 дней | FR-2 |
| 2 | Слот с availableSeats <= 0 показывается, но некликабелен | FR-11 |
| 3 | Если availableSeats < 0 (баг бэкенда) — трактуем как 0 | П. 24 из review |
| 4 | Публичный рейтинг маршала не показывается | FR-18 |

10. Требования (traceability)
    10.1. Functional Requirements
    | ID | Формулировка | Как покрыто |
    | --- | --- | --- |
    | FR-2 | Расписание на 7 дней + фильтр | Блок переключателя дат + BS-003 |
    | FR-6 | Две цены на карточке | SlotCard |
    | FR-11 | Empty state «мест нет» | Состояние Empty + некликабельная карточка |
    | FR-18 | Нет публичного рейтинга | Нет блока рейтинга |

10.2. Non-Functional Requirements
| ID | Формулировка | Как обеспечено |
| --- | --- | --- |
| NFR-1 | Бэкенд — источник истины | Только GET /slots |
| NFR-6 | Нет групповых броней / листа ожидания | Лист ожидания не предлагаем |

11. Критерии приёмки (DoD)
- [ ] Горизонтальный свайп по дням работает.
- [ ] Pull-to-refresh обновляет список.
- [ ] Слот без мест некликабелен.
- [ ] Empty states реализованы.
- [ ] Таймаут 10 с → Error.

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Бэкенд вернул availableSeats < 0 | Трактуем как 0 (п. 24 из review) |
    | 2 | Слот со статусом CancelledByCenter | Карточка с лейблом «Отменён центром», некликабельна (п. 25 из review) |
    | 3 | Сеть упала при смене даты | Оставить предыдущий диапазон, toast «Не удалось обновить» (п. 20 из review) |
    | 4 | На выбранную дату нет слотов после BS-003 | Inline-предупреждение (п. 19 из review) |

12.1. Явные ограничения скоупа
- Лист ожидания — вне скоупа (NFR-6, FR-11).
- Публичный рейтинг маршала — вне скоупа (FR-18).
- Визуальная карта маршрута — вне скоупа (NFR-8).
