3. Назначение и контекст
   3.1. Краткое описание
   Success-шторка, которая показывается клиенту после успешной отправки оценки маршалу (1–5 звёзд). Финальный шаг сценария UC-7.

3.2. Пользовательская цель
Клиент хочет получить подтверждение, что его оценка принята, и вернуться к приложению.

3.3. Бизнес-ценность
Позитивное подкрепление → повышение вероятности повторных оценок в будущем → улучшение качества сервиса через внутреннюю аналитику.

3.4. Контекст открытия
Сессия: авторизован
Сеть: обязательна (только что выполнен POST /slots/{slotId}/ratings → 201)
Предшествующее действие: успешная отправка оценки на SCR-008

8. Логика и взаимодействия
   8.1. Инициализация
1. Получить MarshalRating из SCR-008 (response 201).
2. Отрисовать контент.
3. (Опционально) запустить таймер автозакрытия 3 с.

8.2. Happy path
1. Оценка отправлена → открывается BS-006.
2. Пользователь видит благодарность.
3. Тапает «Закрыть» / свайпает вниз / ждёт автозакрытия.
4. Возврат на SCR-005 (если открыли из SCR-006) или закрытие модалки (если открыли по push).

12. Edge cases и ограничения
    | # | Сценарий | Поведение |
    | --- | --- | --- |
    | 1 | Push-дубликат: клиент уже оценил, но push пришёл снова | На уровне SCR-008 — проверка `rated` → тост «Вы уже оценили этот заезд» + закрытие; BS-006 не открывается (п. 11 из review) |
    | 2 | Заезд отменён центром | Push не приходит (п. 12 из review); BS-006 не открывается |
    | 3 | Сеть упала сразу после 201 (редкий race) | Шторка уже открыта — данные локально сохранены из response; флаг `rated` на бэкенде установлен |
    | 4 | Клиент быстро нажимает «Закрыть» до отрисовки | Шторка закрывается; оценка уже сохранена на бэкенде |

12.1. Явные ограничения скоупа
- Текстовые комментарии — вне скоупа (FR-14).
- Публичный рейтинг — вне скоупа (FR-18).
- Автозакрытие через 3 с — опционально, не блокирующий UX.
