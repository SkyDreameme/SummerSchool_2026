# Нефункциональные требования (NFR)

> Ограничения, качества системы, интеграции и явные «вне скоупа». Не описывают поведение, а определяют рамки.
>
Источник: [`domain-description.md`](../1-elicitation/domain-description.md)

| ID | Формулировка | Приоритет | Источник |
|---|---|---|---|
| NFR-1 | Бэкенд является black-box источником истины; приложение обязано полагаться на его ответы и корректно обрабатывать отказы (включая «мест нет»). | Must | §2 «Границы системы» |
| NFR-2 | Атомарная проверка свободных мест при бронировании — ответственность бэкенда, а не клиента. | Must | §2, §6.1 |
| NFR-3 | Push-уведомления должны доставляться через APNs / FCM; триггер отправки — на стороне бэкенда. | Must | §7 «Интеграции», §6.5 |
| NFR-4 | SMS-шлюз для отправки кодов авторизации предоставляется и обслуживается бэкендом. | Must | §7 |
| NFR-5 | Приложение не должно реализовывать онлайн-эквайринг (карты, СБП внутри приложения) в рамках текущей поставки. | Must (ограничение) | §8 «Состояния, не покрываемые текущей поставкой» |
| NFR-6 | Приложение не должно реализовывать групповые брони и лист ожидания. | Must (ограничение) | §8, §6.4 |
| NFR-7 | Приложение не должно реализовывать waivers / цифровые согласия / загрузку документов / возрастные и медицинские ограничения. | Must (ограничение) | §8, ответ на вопрос №3 |
| NFR-8 | Приложение не должно выполнять валидацию «опытности» клиента при выборе конфигурации трассы. | Must (ограничение) | §4.2, ответ на вопрос №1 |
| NFR-9 | Приложение не должно реализовывать автоматические компенсации и возвраты средств при отмене заезда центром. | Must (ограничение) | §6.3, ответ на вопрос №11 |
| NFR-10 | Интерфейсы ролей «Маршал» и «Владелец/Администратор» находятся вне скоупа текущей поставки. | Must (ограничение) | §2, §3 «Роли» |
| NFR-11 | Логика начисления статусов лояльности находится на стороне бэкенда; приложение только отображает и применяет скидку. | Must | §6.6 |
| NFR-12 | Переход брони в статус `Состоялась` происходит автоматически по факту прошедшего времени окончания заезда (без участия клиента). | Must | §5 «Правила переходов» |
