# Варианты использования (UC)

> Последовательности шагов взаимодействия актёра (клиента) с системой для достижения конкретной цели.
>
Источник: [`domain-description.md`](../1-elicitation/domain-description.md)

| ID | Название | Краткое описание | Приоритет | Источник |
|---|---|---|---|---|
| UC-1 | Регистрация / вход по SMS | Клиент вводит номер телефона → получает SMS-код → вводит код → получает доступ к приложению. | Must | §6.7, §7 |
| UC-2 | Просмотр расписания | Клиент открывает экран расписания, видит слоты на 7 дней, при необходимости расширяет диапазон фильтром дат; видит доступные места, конфигурацию, цены. | Must | §6.1, §4.3 |
| UC-3 | Создание брони | Клиент выбирает слот → выбирает конфигурацию трассы (Short/Long) и экипировку (Rental/Own) → приложение создаёт бронь (1 клиент = 1 бронь) → получает `depositLink`. | Must | §4.4, §6.1 |
| UC-4 | Оплата депозита | Клиент переходит по `depositLink`, выполняет перевод по номеру телефона; бэкенд фиксирует оплату и переводит бронь в статус `Paid`. | Must | §4.4, §6.2 |
| UC-5 | Отмена брони клиентом | Клиент нажимает «Отменить» в активной брони; приложение проверяет условие `now + 2h ≤ startsAt` → при выполнении отправляет запрос на отмену; при нарушении — кнопка неактивна. | Must | §6.3, §5 |
| UC-6 | Отмена заезда центром | Бэкенд инициирует отмену (погода и т.п.) → бронь переходит в статус `CancelledByCenter` с заполненным `cancellationReason` → клиент получает push. | Must | §4.3, §6.3, §5 |
| UC-7 | Оценка маршала | Через 1 час после окончания заезда клиент получает push → открывает экран/модалку → ставит оценку от 1 до 5 звёзд (без текста) → оценка отправляется на бэкенд. | Should | §4.6, §6.5 |
| UC-8 | Просмотр профиля и лояльности | Клиент открывает профиль → видит номер телефона, статус лояльности (`None` / `Bronze` / `Silver`) и размер скидки (0% / 10% / 20%). | Should | §4.1, §6.6 |
| UC-9 | Просмотр истории заездов | Клиент открывает раздел «Мои заезды» → видит три вкладки/состояния: активные брони, состоявшиеся заезды, отменённые заезды. | Must | §6.9 |
| UC-10 | Получение push-напоминания | Клиент получает push за 24 часа и за 2 часа до старта заезда; push информационный, без action-кнопок. | Should | §6.5 |
| UC-11 | Обработка отсутствия мест | Клиент выбирает слот без свободных мест → видит empty state «мест нет»; лист ожидания не предлагается. | Must | §6.4, ответ на вопрос №7 |
