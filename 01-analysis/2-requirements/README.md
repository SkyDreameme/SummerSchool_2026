## Сводка по приоритетам

- **Must** — ядро поставки: бронирование, оплата депозитом, отмены, SMS-вход, история заездов, уведомления об отмене центром. Без этих требований цель поставки (§1) не достигается.
- **Should** — ценность, но не блокирует основной сценарий: программа лояльности, оценки маршалов, push-напоминания.
- **Won't** (в рамках текущей поставки) — зафиксировано в NFR как ограничения: онлайн-эквайринг, групповые брони, лист ожидания, waivers, публичные рейтинги маршалов, автоматические компенсации (§8).
