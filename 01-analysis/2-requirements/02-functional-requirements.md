# Функциональные требования (FR)

> Поведенческие требования: что именно система должна делать для пользователя.
>
Источник: [`domain-description.md`](../1-elicitation/domain-description.md)

| ID | Формулировка | Приоритет | Источник |
|---|---|---|---|
| FR-1 | Система должна обеспечивать регистрацию и вход клиента исключительно по номеру телефона через SMS-код. | Must | §6.7, ответ на вопрос №13 |
| FR-2 | Система должна отображать расписание слотов на 7 дней вперёд по умолчанию, с возможностью расширения горизонта через фильтр дат. | Must | §6.1 «Бронирование» |
| FR-3 | Система должна позволять клиенту выбрать конфигурацию трассы (Short / Long) при бронировании, без валидации опыта. | Must | §4.2, ответ на вопрос №1 |
| FR-4 | Система должна позволять клиенту выбрать вариант экипировки (Rental / Own) при бронировании. | Must | §4.4 «Бронь», §6.1 |
| FR-5 | Система должна создавать ровно одну бронь для одного клиента в одном заезде (групповые брони не поддерживаются). | Must | §4.4, ответ на вопрос №2 |
| FR-6 | Система должна отображать для каждого слота две цены — `priceWithRental` и `priceOwnGear`, получаемые от бэкенда. | Must | §4.3, §6.1 |
| FR-7 | Система должна автоматически применять скидку лояльности (0% / 10% / 20%) к выбранной цене при расчёте `finalPrice`. | Should | §6.1, §6.6 |
| FR-8 | Система должна генерировать `depositLink` (ссылку на перевод по номеру телефона) после создания брони. | Must | §4.4, §6.2 |
| FR-9 | Система должна отображать статус брони: `Pending` / `Paid` / `Expired`, а также жизненный цикл `Создана → Подтверждена → Состоялась / Отменена`. | Must | §4.4, §5 |
| FR-10 | Система должна позволять клиенту отменить бронь; кнопка «Отменить» должна быть активна только при условии `now + 2h ≤ slot.startsAt`. | Must | §6.3, ответ на вопрос №6 |
| FR-11 | Система должна отображать empty state «мест нет» для слотов без свободных мест; лист ожидания не реализуется. | Must | §6.4, ответ на вопрос №7 |
| FR-12 | Система должна отправлять/получать push-уведомления: за 24 часа и за 2 часа до старта, через 1 час после окончания заезда. | Should | §6.5, ответ на вопрос №10 |
| FR-13 | Push-уведомления должны быть информационными, без action-кнопок «Я буду» / «Отменить». | Should | §6.5, ответ на вопрос №10 |
| FR-14 | Система должна отображать форму оценки маршала (1–5 звёзд) без поля для текстового комментария. | Should | §4.6, ответ на вопрос №9 |
| FR-15 | Система должна отображать в профиле клиента текущий статус лояльности (`None` / `Bronze` / `Silver`) и размер скидки. | Should | §4.1, §6.6 |
| FR-16 | Система должна содержать раздел «Мои заезды» с тремя состояниями: активные брони / состоявшиеся / отменённые. | Must | §6.9, ответ на вопрос №14 |
| FR-17 | При отмене заезда центром система должна отображать статус `CancelledByCenter` и поле `cancellationReason`; автоматические компенсации не реализуются. | Must | §4.3, §6.3, ответ на вопрос №11 |
| FR-18 | Система не должна отображать публичный рейтинг маршала в карточках слотов (оценки — только для внутренней аналитики). | Must | §4.5, ответ на вопрос №8 |
