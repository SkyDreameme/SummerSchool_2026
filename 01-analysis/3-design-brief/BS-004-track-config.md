# BS-004: Конфигурация маршрута

## Пользовательская цель
Получить подробное текстовое описание конфигурации трассы (Short / Long) без визуальной карты.

## Контекст и триггер открытия
- **Родительский экран:** SCR-003 «Детали слота» (также может открываться из карточки слота на SCR-002).
- **Триггер:** тап по иконке информации «ⓘ» рядом с названием конфигурации трассы или по самому лейблу «Short / Long».
- **Тип шторки:** информационная (informational sheet).
- **Важно:** визуальная карта маршрута **вне скоупа** (NFR-8) — шторка содержит только текст.

## Контент и данные на шторке
- Заголовок: «Конфигурация трассы».
- Таб / переключатель: **Short** / **Long**.
- Для каждой конфигурации:
    - Длина трассы (в метрах, если бэкенд отдаёт).
    - Примерное время круга.
    - Текстовое описание: количество поворотов, характерные особенности, рекомендуемый уровень (без валидации опыта клиента — NFR-8).
    - Дополнительные материалы: текстовые примечания от центра.
- Закрытие: свайп вниз / тап по фону / кнопка «Закрыть».

## Связанные требования

### User Stories
- **US-3** — Как клиент, я хочу выбрать конфигурацию трассы (короткую или длинную) при бронировании, чтобы кататься в соответствии со своими предпочтениями.

### Use Cases
- **UC-2** — Просмотр расписания (продолжение: клиент уточняет детали конфигурации перед бронированием).
- **UC-3** — Создание брони (предварительный этап: выбор конфигурации на основе описания).

### Functional Requirements
- **FR-3** — Система должна позволять клиенту выбрать конфигурацию трассы (Short / Long) при бронировании, без валидации опыта.

### Non-Functional Requirements
- **NFR-8** — Приложение не должно выполнять валидацию «опытности» клиента и не отображает визуальную карту маршрута — только текстовое описание.
