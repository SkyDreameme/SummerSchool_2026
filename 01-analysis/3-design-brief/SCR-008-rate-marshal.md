# SCR-008: Оценка маршала

## Пользовательская цель
Быстро оценить работу маршала-инструктора после заезда, не заполняя сложных форм.

## Контент и данные на экране
- Заголовок: «Оцените работу маршала».
- Дата и время заезда (для контекста).
- Шкала оценки: 5 звёзд (тап по звезде фиксирует значение 1–5).
- CTA «Отправить оценку».
- Экран успеха после отправки.
- Поле для текстового комментария **отсутствует** (намеренно).

## Связанные требования

### User Stories
- **US-10** — Как клиент, я хочу получить push с запросом оценки маршала через 1 час после окончания заезда, чтобы поделиться впечатлением.

### Use Cases
- **UC-7** — Оценка маршала: через 1 час после окончания заезда клиент получает push → открывает экран/модалку → ставит оценку 1–5 звёзд (без текста) → оценка отправляется.

### Functional Requirements
- **FR-12** — Система должна отправлять/получать push-уведомления через 1 час после окончания заезда.
- **FR-13** — Push-уведомления должны быть информационными, без action-кнопок.
- **FR-14** — Система должна отображать форму оценки маршала (1–5 звёзд) без поля для текстового комментария.

### Non-Functional Requirements
- **NFR-3** — Push-уведомления доставляются через APNs / FCM; триггер — на стороне бэкенда.
