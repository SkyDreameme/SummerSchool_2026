# BS-006: Успешная оценка маршала

## Пользовательская цель
Поблагодарить клиента за обратную связь и закрыть форму оценки.

## Контекст и триггер открытия
- **Родительский экран:** SCR-008 «Оценка маршала».
- **Триггер:** успешный ответ бэкенда на запрос отправки оценки (1–5 звёзд).
- **Тип шторки:** success-уведомление.

## Контент и данные на шторке
- Иконка благодарности (сердце / звезда).
- Заголовок: «Спасибо за оценку!»
- Краткое описание: «Ваш отзыв поможет нам стать лучше.»
- Кнопка «Закрыть» — закрывает шторку и возвращает на SCR-005 (или закрывает модалку, если оценка открыта по push).

## Связанные требования

### User Stories
- **US-10** — Как клиент, я хочу получить push с запросом оценки маршала через 1 час после окончания заезда, чтобы поделиться впечатлением.

### Use Cases
- **UC-7** — Оценка маршала: завершающий этап — подтверждение успешной отправки оценки.

### Functional Requirements
- **FR-12** — Система должна отправлять/получать push-уведомления, в том числе через 1 час после окончания заезда.
- **FR-14** — Система должна отображать форму оценки маршала (1–5 звёзд) без поля для текстового комментария.

### Non-Functional Requirements
- **NFR-3** — Push-уведомления доставляются через APNs / FCM; триггер — на стороне бэкенда.
