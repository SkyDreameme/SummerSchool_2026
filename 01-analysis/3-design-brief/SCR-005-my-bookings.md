# SCR-005: Мои записи

## Пользовательская цель
Просмотреть историю своих заездов и текущие активные брони в едином разделе.

## Контент и данные на экране
- Три вкладки / состояния:
    - **Активные** — брони в статусах `Pending` / `Paid`.
    - **Состоявшиеся** — заезды, время окончания которых уже прошло (автоматический переход по NFR-12).
    - **Отменённые** — отменённые клиентом или центром (`CancelledByCenter`).
- Для каждой записи: дата, время, конфигурация трассы, статус (цветовой индикатор).
- Empty state для каждой вкладки.
- Pull-to-refresh, состояния загрузки и ошибки.

## Связанные требования

### User Stories
- **US-12** — Как клиент, я хочу видеть историю своих заездов (активные, состоявшиеся, отменённые) в разделе «Мои заезды», чтобы отслеживать свои поездки.

### Use Cases
- **UC-9** — Просмотр истории заездов: клиент открывает раздел и видит три вкладки/состояния.

### Functional Requirements
- **FR-9** — Система должна отображать статус брони: `Pending` / `Paid` / `Expired`, а также жизненный цикл.
- **FR-16** — Система должна содержать раздел «Мои заезды» с тремя состояниями: активные брони / состоявшиеся / отменённые.

### Non-Functional Requirements
- **NFR-12** — Переход брони в статус «Состоялась» происходит автоматически по факту прошедшего времени окончания заезда.
