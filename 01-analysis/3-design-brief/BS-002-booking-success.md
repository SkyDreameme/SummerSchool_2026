# BS-002: Успешное бронирование / Оплата депозита

## Пользовательская цель
Получить ссылку на оплату депозита (`depositLink`) и инструкцию по переводу сразу после создания брони.

## Контекст и триггер открытия
- **Родительский экран:** SCR-004 «Бронирование».
- **Триггер:** успешный ответ бэкенда на запрос создания брони (статус `Pending`).
- **Тип шторки:** success + CTA (переход во внешний банк / СБП).

## Контент и данные на шторке
- Иконка успеха (галочка).
- Заголовок: «Бронь создана».
- Краткая сводка: дата, время, конфигурация трассы, экипировка (Rental / Own), итоговая цена `finalPrice`.
- Блок с `depositLink`:
    - Сумма депозита.
    - Кнопка «Оплатить депозит» → открывает внешнюю ссылку (переход в банк / СБП).
    - Опционально: кнопка «Скопировать ссылку» (для ручной отправки в мессенджер).
- Инструкция: «Переведите сумму по ссылке, чтобы подтвердить бронь. Бронь автоматически перейдёт в статус `Paid` после поступления оплаты.»
- Кнопка «Понятно» / «К моим записям» — закрывает шторку и переводит на SCR-005.

## Связанные требования

### User Stories
- **US-5** — Как клиент, я хочу забронировать одно место в выбранном заезде, чтобы гарантировать себе участие.
- **US-6** — Как клиент, я хочу оплатить депозит по ссылке на перевод по номеру телефона, чтобы подтвердить свою бронь.

### Use Cases
- **UC-3** — Создание брони: клиент выбирает слот → конфигурацию и экипировку → приложение создаёт бронь → получает `depositLink`.
- **UC-4** — Оплата депозита: клиент переходит по `depositLink`, выполняет перевод; бэкенд фиксирует оплату и переводит бронь в статус `Paid`.

### Functional Requirements
- **FR-5** — Система должна создавать ровно одну бронь для одного клиента в одном заезде.
- **FR-8** — Система должна генерировать `depositLink` после создания брони.
- **FR-9** — Система должна отображать статус брони: `Pending` → `Paid` → `Expired`.

### Non-Functional Requirements
- **NFR-2** — Атомарная проверка свободных мест при бронировании — ответственность бэкенда.
- **NFR-5** — Приложение не реализует онлайн-эквайринг; только переход по внешней ссылке.
