# SCR-001: Регистрация / Вход

## Пользовательская цель
Быстро войти в приложение по номеру телефона через SMS-код, не придумывая и не запоминая пароль.

## Контент и данные на экране
- Поле ввода номера телефона с маской и кодом страны.
- Кнопка «Получить SMS-код».
- Экран ввода SMS-кода (4–6 цифр, автоподтверждение при вводе).
- Таймер повторной отправки кода.
- Inline-ошибки: «Некорректный номер», «Неверный код», «Превышено число попыток».
- Ссылка на пользовательское соглашение (опционально, без загрузки документов — см. NFR-7).

## Связанные требования

### User Stories
- **US-1** — Как клиент, я хочу зарегистрироваться и войти в приложение по номеру телефона через SMS-код, чтобы не запоминать пароли.

### Use Cases
- **UC-1** — Регистрация / вход по SMS: клиент вводит номер → получает SMS-код → вводит код → получает доступ.

### Functional Requirements
- **FR-1** — Система должна обеспечивать регистрацию и вход клиента исключительно по номеру телефона через SMS-код.

### Non-Functional Requirements
- **NFR-4** — SMS-шлюз предоставляется и обслуживается бэкендом.
- **NFR-7** — Waivers и цифровые согласия не реализуются.
