# SCR-007: Профиль клиента

## Пользовательская цель
Увидеть свой номер телефона, текущий статус лояльности и размер персональной скидки.

## Контент и данные на экране
- Номер телефона (маскированный или полный, в зависимости от политики).
- Статус лояльности: `None` / `Bronze` / `Silver`.
- Размер скидки: 0% / 10% / 20%.
- Описание условий перехода на следующий уровень (опционально, текстом).
- CTA «Выйти» (сброс сессии).

## Связанные требования

### User Stories
- **US-11** — Как клиент, я хочу видеть свой текущий статус лояльности и размер скидки в профиле, чтобы понимать свой уровень.

### Use Cases
- **UC-8** — Просмотр профиля и лояльности: клиент открывает профиль → видит номер телефона, статус лояльности и размер скидки.

### Functional Requirements
- **FR-15** — Система должна отображать в профиле клиента текущий статус лояльности (None / Bronze / Silver) и размер скидки.

### Non-Functional Requirements
- **NFR-11** — Логика начисления статусов лояльности находится на стороне бэкенда; приложение только отображает и применяет скидку.
