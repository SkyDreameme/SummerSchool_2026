# SCR-004: Бронирование

## Пользовательская цель
Подтвердить выбор конфигурации трассы и экипировки, создать бронь и получить ссылку на оплату депозита.

## Контент и данные на экране
- Сводка брони: дата, время, конфигурация трассы (Short / Long).
- Селектор экипировки: Rental / Own (влияет на цену).
- Итоговая цена `finalPrice` с применённой скидкой.
- CTA «Забронировать место» (1 клиент = 1 бронь).
- После успешного создания брони:
    - Статус `Pending`.
    - Блок с `depositLink` и CTA «Оплатить депозит» (переход во внешний банк / СБП).
    - Инструкция: «Переведите сумму по ссылке, чтобы подтвердить бронь».
- Состояния ошибки: «мест нет» (race condition), «бронь уже существует», сетевая ошибка.

## Связанные требования

### User Stories
- **US-4** — Как клиент, я хочу выбрать вариант экипировки (прокатная / своя), чтобы повлиять на итоговую цену заезда.
- **US-5** — Как клиент, я хочу забронировать одно место в выбранном заезде, чтобы гарантировать себе участие.
- **US-6** — Как клиент, я хочу оплатить депозит по ссылке на перевод по номеру телефона, чтобы подтвердить свою бронь.

### Use Cases
- **UC-3** — Создание брони: клиент выбирает слот → конфигурацию и экипировку → приложение создаёт бронь → получает `depositLink`.
- **UC-4** — Оплата депозита: клиент переходит по `depositLink`, выполняет перевод; бэкенд фиксирует оплату и переводит бронь в статус `Paid`.

### Functional Requirements
- **FR-3** — Система должна позволять клиенту выбрать конфигурацию трассы (Short / Long) при бронировании.
- **FR-4** — Система должна позволять клиенту выбрать вариант экипировки (Rental / Own) при бронировании.
- **FR-5** — Система должна создавать ровно одну бронь для одного клиента в одном заезде.
- **FR-7** — Система должна автоматически применять скидку лояльности к выбранной цене.
- **FR-8** — Система должна генерировать `depositLink` после создания брони.

### Non-Functional Requirements
- **NFR-2** — Атомарная проверка свободных мест при бронировании — ответственность бэкенда.
- **NFR-5** — Приложение не должно реализовывать онлайн-эквайринг; только переход по внешней ссылке.
- **NFR-6** — Групповые брони не поддерживаются.
