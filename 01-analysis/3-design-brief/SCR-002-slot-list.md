# SCR-002: Расписание слотов

## Пользовательская цель
Просмотреть доступные слоты на ближайшие дни и выбрать удобное время заезда.

## Контент и данные на экране
- Горизонтальный переключатель дат (по умолчанию 7 дней вперёд).
- Фильтр «Расширить диапазон дат» (календарь).
- Список карточек слотов, для каждой:
    - Дата и время начала (`startsAt`).
    - Конфигурация трассы (Short / Long).
    - Две цены: `priceWithRental` и `priceOwnGear`.
    - Индикатор свободных мест.
- Empty state «Мест нет» (без листа ожидания).
- Состояния загрузки и ошибки сети.

## Связанные требования

### User Stories
- **US-2** — Как клиент, я хочу видеть расписание свободных слотов на 7 дней вперёд, чтобы выбрать удобное время заезда.
- **US-14** — Как клиент, я хочу видеть, что мест в слоте нет (empty state), чтобы не пытаться забронировать недоступный заезд.

### Use Cases
- **UC-2** — Просмотр расписания: клиент открывает экран, видит слоты на 7 дней, при необходимости расширяет диапазон фильтром дат; видит доступные места, конфигурацию, цены.
- **UC-11** — Обработка отсутствия мест: клиент видит empty state «мест нет»; лист ожидания не предлагается.

### Functional Requirements
- **FR-2** — Система должна отображать расписание слотов на 7 дней вперёд по умолчанию, с возможностью расширения горизонта через фильтр дат.
- **FR-6** — Система должна отображать для каждого слота две цены — `priceWithRental` и `priceOwnGear`.
- **FR-11** — Система должна отображать empty state «мест нет» для слотов без свободных мест; лист ожидания не реализуется.
- **FR-18** — Система не должна отображать публичный рейтинг маршала в карточках слотов.

### Non-Functional Requirements
- **NFR-1** — Бэкенд является black-box источником истины; приложение корректно обрабатывает ответы, включая «мест нет».
- **NFR-6** — Групповые брони и лист ожидания не реализуются.
