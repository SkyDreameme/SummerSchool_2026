# SCR-006: Детали брони

## Пользовательская цель
Просмотреть полную информацию о конкретном заезде и при необходимости отменить его или узнать причину отмены центром.

## Контент и данные на экране
- Дата, время начала (`startsAt`) и окончания заезда.
- Конфигурация трассы (Short / Long).
- Выбранная экипировка (Rental / Own) и итоговая цена.
- Текущий статус брони:
    - `Pending` — ожидает оплаты депозита.
    - `Paid` — депозит оплачен, заезд подтверждён.
    - `Expired` — время оплаты истекло.
    - `CancelledByCenter` — отменено центром.
- Поле `cancellationReason` (если статус `CancelledByCenter`).
- CTA «Отменить»:
    - Активна только при условии `now + 2h ≤ startsAt`.
    - Неактивна/скрыта иначе.
- CTA «Оценить маршала» (появляется после завершения заезда, по push-триггеру).
- Состояния загрузки и ошибки.

## Связанные требования

### User Stories
- **US-8** — Как клиент, я хочу отменить бронь не позднее чем за 2 часа до старта, чтобы освободить место и не потерять деньги.
- **US-13** — Как клиент, я хочу узнать причину отмены заезда центром, чтобы понимать, почему заезд не состоялся.

### Use Cases
- **UC-5** — Отмена брони клиентом: клиент нажимает «Отменить» → приложение проверяет условие `now + 2h ≤ startsAt` → отправляет запрос.
- **UC-6** — Отмена заезда центром: бронь переходит в статус `CancelledByCenter` с заполненным `cancellationReason` → клиент получает push.

### Functional Requirements
- **FR-9** — Система должна отображать статус брони и жизненный цикл.
- **FR-10** — Система должна позволять клиенту отменить бронь; кнопка активна только при `now + 2h ≤ slot.startsAt`.
- **FR-17** — При отмене заезда центром система должна отображать статус `CancelledByCenter` и поле `cancellationReason`.

### Non-Functional Requirements
- **NFR-9** — Приложение не должно реализовывать автоматические компенсации и возвраты средств при отмене центром.
