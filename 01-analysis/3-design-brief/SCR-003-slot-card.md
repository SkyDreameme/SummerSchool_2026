# SCR-003: Детали слота

## Пользовательская цель
Познакомиться с деталями выбранного слота и увидеть итоговую цену с учётом своей скидки лояльности.

## Контент и данные на экране
- Дата и время начала заезда (`startsAt`).
- Конфигурация трассы: Short / Long (без валидации опыта клиента).
- Две цены: `priceWithRental` и `priceOwnGear`.
- Блок «Итоговая цена» с автоматическим применением скидки лояльности (0% / 10% / 20%) → `finalPrice`.
- Индикатор текущего статуса лояльности клиента (None / Bronze / Silver).
- CTA «Забронировать» (неактивна, если мест нет).
- Состояния загрузки и ошибки.

## Связанные требования

### User Stories
- **US-3** — Как клиент, я хочу выбрать конфигурацию трассы (короткую или длинную) при бронировании, чтобы кататься в соответствии со своими предпочтениями.
- **US-7** — Как клиент, я хочу видеть итоговую цену с учётом моей скидки лояльности, чтобы понимать, сколько я заплачу.

### Use Cases
- **UC-2** — Просмотр расписания (продолжение: переход к деталям слота).

### Functional Requirements
- **FR-3** — Система должна позволять клиенту выбрать конфигурацию трассы (Short / Long) при бронировании, без валидации опыта.
- **FR-6** — Система должна отображать для каждого слота две цены — `priceWithRental` и `priceOwnGear`.
- **FR-7** — Система должна автоматически применять скидку лояльности (0% / 10% / 20%) к выбранной цене при расчёте `finalPrice`.

### Non-Functional Requirements
- **NFR-8** — Приложение не должно выполнять валидацию «опытности» клиента при выборе конфигурации трассы.
- **NFR-11** — Логика начисления статусов лояльности находится на стороне бэкенда; приложение только отображает и применяет скидку.
