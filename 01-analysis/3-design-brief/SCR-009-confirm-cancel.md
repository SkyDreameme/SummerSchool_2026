# SCR-009: Подтверждение отмены

## Пользовательская цель
Подтвердить намерение отменить бронь, избежав случайного нажатия.

## Контент и данные на экране
- Модальное окно / bottom sheet.
- Заголовок: «Отменить бронь?»
- Описание: дата, время заезда, предупреждение о невозможности восстановления.
- Два действия:
    - «Отменить бронь» (деструктивное, подтверждает действие).
    - «Оставить бронь» (закрывает модалку).
- Состояние загрузки во время отправки запроса.
- Ошибка при неуспешной отмене (например, истекло окно 2 часов).

## Связанные требования

### User Stories
- **US-8** — Как клиент, я хочу отменить бронь не позднее чем за 2 часа до старта.

### Use Cases
- **UC-5** — Отмена брони клиентом: приложение проверяет условие `now + 2h ≤ startsAt` → при выполнении отправляет запрос на отмену.

### Functional Requirements
- **FR-10** — Система должна позволять клиенту отменить бронь; кнопка «Отменить» активна только при `now + 2h ≤ slot.startsAt`.

### Non-Functional Requirements
- **NFR-1** — Бэкенд является источником истины; приложение корректно обрабатывает отказы (например, если окно отмены уже закрыто на стороне сервера).
