**Версия:** 1.0  
**Дата:** 2026-07-05  
**Стек:** Kotlin, Jetpack Compose, DataStore, Coroutines  
**Тип приложения:** Android (мобильное)  
**Состояние:** ✅ Все пользовательские сценарии реализованы (прототип с мок-бэкендом)

---

## 📌 Общее описание

Приложение «Апекс» — это клиентская часть для картинг-центра. Оно позволяет пользователям:
- входить по SMS-коду,
- просматривать расписание заездов,
- бронировать места с выбором экипировки,
- оплачивать депозит по ссылке (симуляция),
- управлять своими бронями (просмотр, отмена),
- оценивать маршалов после заезда,
- просматривать профиль с уровнем лояльности.

Все данные хранятся в памяти (мок-репозиторий), сессия сохраняется в DataStore.

## 🧩 Функциональные блоки

### 1. Авторизация (SCR-001)

- [x] Вход по номеру телефона с маской +7 (валидация E.164).
- [x] Отправка SMS-кода с таймером повторной отправки (60 сек).
- [x] Ввод кода (4–6 цифр) с автоподтверждением.
- [x] Демо-код `1234` для быстрого входа.
- [x] Обработка ошибок:
    - неверный номер,
    - неверный код (счётчик попыток),
    - превышение попыток → кнопка «Связаться с поддержкой».
- [x] Повторный вход существующего номера (не создаётся новый клиент).
- [x] Сохранение JWT в DataStore (автовход при повторном запуске).
- [x] Выход из приложения (очистка сессии, переход на экран входа).

---

### 2. Расписание слотов (SCR-002)

- [x] Отображение слотов на 7 дней (горизонтальный переключатель дат).
- [x] Карточка слота:
    - время старта,
    - конфигурация трассы (Short/Long) с информационной иконкой,
    - имя маршала (без публичного рейтинга),
    - две цены (с прокатной экипировкой / со своей),
    - индикатор свободных мест (`availableSeats / totalSeats`).
- [x] Некликабельные слоты при `availableSeats = 0` (лейбл «Мест нет»).
- [x] Слоты, отменённые центром, помечаются как «Отменён центром» и некликабельны.
- [x] Расширение диапазона дат через календарь (BS-003) — выбор произвольного периода.
- [x] Обновление списка по кнопке в панели (замена pull-to-refresh).
- [x] Состояния: Loading (скелетон), Error (с повторной попыткой), Empty («Нет доступных заездов»).

---

### 3. Детали слота (SCR-003)

- [x] Полная информация: дата, время, конфигурация, маршал.
- [x] Отображение двух базовых цен.
- [x] Итоговая цена с учётом скидки лояльности (0%, 10%, 20%).
- [x] Бейдж статуса лояльности (None / Bronze / Silver).
- [x] Кнопка «Забронировать»:
    - активна при наличии мест и открытом статусе,
    - disabled при отсутствии мест или отмене центром (с показом причины).
- [x] Информационная шторка о конфигурации трассы (BS-004) — текстовое описание.
- [x] Fallback для неизвестного статуса лояльности («Нет статуса»).

---

### 4. Бронирование и оплата депозита (SCR-004, BS-002)

- [x] Выбор экипировки: «Прокат» / «Своя» (сегмент-контрол).
- [x] Пересчёт итоговой цены при смене экипировки.
- [x] Создание брони с проверкой:
    - наличие мест,
    - отсутствие активной брони у пользователя (FR-5).
- [x] Обработка ошибок:
    - `409 NO_SEATS` → inline-ошибка + CTA «Выбрать другое время»,
    - `409 BOOKING_ALREADY_EXISTS` → inline-ошибка + CTA «К моим записям»,
    - `410 SLOT_CANCELLED` → inline-ошибка с причиной.
- [x] Success-шторка (BS-002) после создания брони:
    - иконка успеха,
    - сводка брони (дата, время, конфигурация, экипировка, цена),
    - ссылка на оплату (`depositLink`),
    - кнопки: «Оплатить депозит» (открывает внешний URL), «Скопировать ссылку»,
    - инструкция на случай задержки оплаты.
- [x] Переход в «Мои записи» (вкладка «Активные») после бронирования.

---

### 5. Мои записи (SCR-005)

- [x] Три вкладки: **Активные**, **Состоявшиеся**, **Отменённые**.
- [x] Список броней с карточками:
    - дата, время, конфигурация,
    - статус с цветовым индикатором (Pending, Paid, Expired, Completed, CancelledByClient, CancelledByCenter).
- [x] Empty-состояния для каждой вкладки (заголовок + подсказка).
- [x] Обновление списка по кнопке в панели.
- [x] Переход на экран деталей брони при клике.

---

### 6. Детали брони и отмена (SCR-006, BS-001, BS-005)

- [x] Полная информация: дата, время, конфигурация, экипировка, цена, статус.
- [x] Отображение причины отмены центром (если есть).
- [x] Блок оплаты депозита для статуса `Pending` (показывает ссылку и кнопку оплаты).
- [x] Отмена брони клиентом:
    - кнопка «Отменить» активна при `canCancel == true` (условие `now + 2h ≤ startsAt` включительно),
    - при disabled — тултип «Отмена доступна не позднее чем за 2 часа»,
    - открывается шторка подтверждения (BS-001) с предупреждением о невозвратности,
    - деструктивная кнопка «Отменить бронь» и «Оставить бронь».
- [x] Обработка ошибки `422 CANCEL_WINDOW_CLOSED` — inline-сообщение.
- [x] Success-шторка после отмены (BS-005) с подтверждением и напоминанием о возврате по правилам центра.
- [x] Автоматический переход в `Completed` по истечении времени заезда (в мок-репозитории).
- [x] TTL для `Pending` (30 минут) — автоматический переход в `Expired`.
- [x] Fallback для неизвестных статусов (попадают в «Отменённые»).

---

### 7. Оценка маршала (SCR-008, BS-006)

- [x] Форма оценки: 5 звёзд (без текстового поля, FR-14).
- [x] Открывается только для завершённых заездов (`status == Completed`), только если `now >= endsAt` и `rated == false`.
- [x] Точка входа: кнопка «Оценить маршала» на экране деталей брони.
- [x] Отправка оценки:
    - успех → success-шторка (BS-006) «Спасибо за оценку!»,
    - ошибка `409 ALREADY_RATED` → тост + закрытие,
    - ошибка `422 RATING_NOT_ALLOWED` → тост.
- [x] Повторная оценка блокируется флагом `rated`.

---

### 8. Профиль (SCR-007)

- [x] Отображение номера телефона.
- [x] Отображение статуса лояльности (None / Bronze / Silver) с бейджем и размером скидки.
- [x] Контакты поддержки: кнопки «Позвонить» и «Написать в Telegram».
- [x] Кнопка «Выйти» (с очисткой сессии и переходом на экран входа).
- [x] **Нет** описания условий перехода на следующий уровень (OOS-018, вне скоупа).

---

### 9. Общие UI-компоненты и состояния

- [x] **Loading**: скелетон для списков, спиннер на кнопках.
- [x] **Error**: иллюстрация + заголовок + кнопка «Повторить» (таймаут 10 секунд).
- [x] **Empty**: иллюстрация + заголовок + подсказка (для каждого экрана своя).
- [x] **Success**: тосты или шторки с подтверждением (бронирование, отмена, оценка).
- [x] Единая навигация через `NavHost` (Single Activity), нижняя панель (3 вкладки).
- [x] Семантические цвета для статусов броней (из `00-foundations`).
- [x] Базовые компоненты: `ApexPrimaryButton`, `ApexSecondaryButton`, `ApexDestructiveButton`, `InfoCard`, `StatusBadge`, `LabelValueRow`.

---

### 10. Технические особенности

- [x] **Мок-репозиторий** (`ApexRepository`):
    - генерирует слоты и брони,
    - имитирует задержку сети (400 мс),
    - обрабатывает ошибки API (409, 410, 422),
    - управляет состоянием в `StateFlow`.
- [x] **DataStore** для сохранения JWT и номера телефона (автовход).
- [x] **Тема**: светлая, с адаптивными цветами.
- [x] **Доступность**: тач-таргеты ≥44×44 dp, контрастные цвета.
- [x] **Tone of Voice**: дружелюбные и краткие сообщения (без канцеляризмов).
- [x] **Поддержка нескольких экранов**: портретная ориентация, адаптивные отступы.

## ❌ Out of Scope (не реализовано)

- Онлайн-эквайринг (оплата через внешний URL).
- Групповые брони и лист ожидания.
- Визуальная карта трассы и валидация опыта клиента.
- Автоматический возврат депозита.
- Интерфейсы для маршалов и администраторов.
- Публичный рейтинг маршалов и текстовые комментарии.
- Смена номера телефона и удаление аккаунта.
- Реальное API-подключение (все данные моковые).
